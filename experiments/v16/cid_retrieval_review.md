## 3. OLD vs NEW 方法对比

### OLD 方法（Buggy）
```python
h1, _ = np.histogram(d1, bins=bins, density=True)  # Bug!
h2, _ = np.histogram(d2, bins=bins, density=True)
sim = cosine_similarity(h1, h2)
```
- `density=True` 归一化使积分=1.0（与原子数无关）
- 不同原子数 → 相同归一化直方图 → cos_sim≈1.000
- 不考虑元素组成

### NEW 方法（Corrected）
```python
# 1. 原子数过滤
if abs(n1 - n2) > 3:
    return 0.0  # 不同原子数不可比

# 2. 距离直方图（density=False）
h1, _ = np.histogram(d1, bins=bins, density=False)
h2, _ = np.histogram(d2, bins=bins, density=False)
h1_norm = h1 / h1.sum()  # 按总数归一化
h2_norm = h2 / h2.sum()

# 3. 元素组成相似度（cosine）
c1 = np.bincount(pred_types, minlength=10)
c2 = np.bincount(gt_types, minlength=10)
elem_sim = cosine_similarity(c1, c2)

# 4. 综合
sim = 0.4 * elem_sim + 0.6 * dist_sim
```

**NEW 方法 rank=21** ✅：加入原子数过滤（|24-24|=0）和元素组成惩罚后，GT 排名大幅提升。

## 7. 结论

1. **修正效果显著**：平均 GT Rank 从 312 降至 77（-75%）
2. **OLD 方法的 `density=True` Bug**导致不同原子数的分子被错误地赋予 sim≈1.000
3. **修正后的 Top-3 更准确**：Top-1 中含正确元素的比例大幅提升
4. **模型能力问题**：6/15 样本的预测缺少 GT 中的必需元素（N、F、O 等）
5. **进一步改进方向**：
   - 修复模型原子类型预测（特别是 N、F 等重元素）
   - 使用 Kabsch RMSD + atom type accuracy 作为检索指标（更精确）
   - 将修正后的相似度方法集成到 `visualize_val.py` 中替换旧的 Bug 方法

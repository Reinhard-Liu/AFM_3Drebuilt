## V19.2 方案：对象级 2D 中心、类型、边，再接 Z

### 为什么要从旧的联合训练改到对象级联合训练

当前 V19.1 的联合训练虽然已经把 `2D + z` 放进了同一个模型里，但它的主监督仍然主要是稠密通道图：

- 原子占据图
- 键图
- 类型图
- z 图

这种设计能学出一部分 2D 结构和 z 信息，但它和参考论文、以及当前项目真正想要的“分子对象重建”仍然不完全一致。

当前最明显的问题是：

- 类型图误差下降，不等于原子类型真的学会了
- 键图误差下降，不等于原子之间的连接关系真的学会了
- 稠密图误差会掩盖“对象级”失败

因此，V19.2 要把联合训练的主目标换成：

- 原子中心
- 中心类型
- 中心之间的边
- 这些对象对应的 z

### V19.2 的训练主线

输入：

- 10 层 AFM 图像堆栈

输出：

- 2D 原子中心热图
- GT 中心条件下的原子类型
- GT 中心条件下的中心边关系
- z 图

这里先保留 GT 中心条件的原因是：

- 当前最需要先确认“共享 AFM 特征能不能同时支持对象级类型和边”
- 而不是立即把错误来源混在“中心检测错误”和“类型/边错误”里

### V19.2 的具体结构

1. 共享主干

- 继续使用 `V19JointUNet`
- 主干负责从 AFM 堆栈提取共享局部特征

2. 原子中心图分支

- 继续预测 `atom_map`
- 这个分支回答“哪里有原子”

3. 对象级类型头

- 在每个真实原子中心位置采样共享特征和 AFM 局部高度曲线
- 直接做离散原子类型分类

4. 对象级边头

- 对每一对真实原子中心，预测它们之间是否有边
- 输入包括：
  - 两个中心的共享特征
  - 两个中心的 AFM 局部特征
  - 它们的相对坐标和距离

5. z 图分支

- 继续预测 `z_map`
- 与对象级中心一起构成未来 3D 重建的初值

### V19.2 的主指标

训练和中期验证时，主看这些对象级指标：

- `atom_center_score_r3`
  中文：真实原子中心附近的原子响应强度

- `typed_center_score_r3`
  中文：真实原子中心附近，位置和类型一起对上的软分数

- `center_type_acc_gtcoord`
  中文：真实中心条件下的原子类型准确率

- `center_macro_f1_gtcoord`
  中文：真实中心条件下的原子类型宏平均 F1

- `center_hetero_f1_gtcoord`
  中文：真实中心条件下的杂原子 F1

- `center_edge_f1_gtcoord`
  中文：真实中心条件下的对象级边 F1

- `atom_z_mae_r3`
  中文：真实中心附近的 z 平均偏差

### V19.2 的判断标准

这版不是最终结构重建终点，而是中间验证版。

它的目标是回答：

1. 共享 AFM 编码特征是否能同时支持：
   - 中心检测
   - 对象级类型
   - 对象级边
   - z 轴

2. 在联合训练中，类型教师蒸馏是否能保住类型信息

只有当这些对象级指标都明显上升，才值得再往“预测中心条件”“预测边闭环”“完整 3D”推进。

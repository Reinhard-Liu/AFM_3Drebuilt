## V19 方案：先 2D 分子图，再补 Z 轴与 3D 几何

### 1. 为什么要进入 V19

当前主线的问题不是单纯训练轮次不够，而是任务定义过重：

- 输入是 10 层不同高度的 AFM 图像堆栈
- 输出却要求一次性完成 3D 坐标、原子类型、原子数、环结构、连接关系

这会导致模型在最基础的几件事都没先学会：

- 原子在 XY 平面的大致位置
- 这个位置更像什么原子
- 哪些原子之间存在局部连接

而参考论文《Molecular identification with atomic force microscopy and conditional generative adversarial networks》之所以效果好，是因为它做的是：

- AFM 堆栈 -> 2D 球棍图

也就是先把局部图像翻译成局部化学图形，而不是直接生成整团 3D 分子。

因此 V19 的核心调整是：

- 第一阶段先做 2D 分子图解析
- 第二阶段再根据 AFM 多高度信息补 Z 轴和 3D 几何

### 2. V19 的两阶段目标

#### 阶段 1：AFM 堆栈 -> 2D 分子图/位点图

这一阶段只解决 4 件事：

1. 原子在 XY 平面上大致在哪里
2. 这些位置分别是什么原子类型
3. 哪些位置之间存在局部化学键
4. 哪些位置构成环骨架，哪些位置是侧链连接位点

项目中的具体设计：

- 输入保持不变：10 层 AFM 图像堆栈
- 先训练一个 2D 图解析器，而不是直接训练 3D 生成器
- 输出目标不再是 3D 点云，而是结构化的 2D 通道图

最小实现采用 12 通道目标图：

- 第 1 通道：原子占据图
- 第 2 通道：化学键图
- 第 3-12 通道：10 类原子类型图

对应项目代码设计：

- 新增 2D 目标渲染工具
- 新增 V19 第一阶段训练脚本
- 复用现有 2D U-Net 生成器和 Patch 判别器，但不再输出简单 RGB 图，而是输出结构通道图

#### 阶段 2：2D 分子图 + AFM 堆栈 -> Z 轴与 3D 几何

这一阶段只在第一阶段稳定后进行。

目标：

1. 为每个已经识别出的 2D 原子位点补预测 Z 坐标
2. 让环结构具有合理平面性
3. 让侧链沿正确方向展开
4. 最后再用轻量 3D 细化器做整体几何修正

项目中的具体设计：

- 第一阶段输出的 2D 位点图与连接图，将作为第二阶段的结构先验
- 第二阶段不再从零生成整团原子，而是从 2D 图出发补全 Z
- 原有扩散模型将从“主生成器”降级成“3D 细化器”

### 3. V19 第一阶段的训练指标

V19 第一阶段不再主用严格通过率，而是主用连续误差型指标。

主要监控指标：

- `atom_xy_mae`
  中文：原子二维位置平均偏差
  含义：预测的 2D 原子位置距离真实位置平均差多少像素或归一化距离

- `bond_map_mae`
  中文：键图平均偏差
  含义：预测的键图和真实键图平均差多少

- `type_map_mae`
  中文：原子类型图平均偏差
  含义：10 类原子类型通道图和真实目标图平均差多少

- `typed_center_match`
  中文：类型正确的原子中心匹配度
  含义：既落在正确原子中心附近，又类型正确的位点比例

- `atom_type_macro_f1_2d`
  中文：二维原子类型宏平均 F1
  含义：每类原子分别计算后平均，避免被碳氢掩盖

- `ch_collapse_rate_2d`
  中文：二维碳氢塌缩率
  含义：模型是否把大部分位点都塌成碳/氢

这一阶段不主用：

- 严格连接边 F1
- 严格结构通过率
- 原子总数完全正确率

这些保留到第二阶段后期和最终评估。

### 4. V19 第一阶段在项目中的具体代码设计

#### 4.1 数据与目标

不修改底层数据集输入格式，仍使用：

- `afm_stack`
- `coords`
- `atom_types`
- `atom_mask`

在训练脚本内根据真实坐标和原子类型即时渲染 2D 目标图：

- 使用归一化后的 XY 坐标投影到图像平面
- 依据元素类型渲染原子盘
- 依据近邻距离与共价半径规则渲染键

#### 4.2 模型

第一阶段最小模型：

- 输入：10 层 AFM 堆栈
- 主体：现有 `UNetGenerator`
- 输出：12 通道 2D 化学结构图

对应通道：

- 1 个原子占据通道
- 1 个键图通道
- 10 个元素类型通道

判别器：

- 复用现有 `PatchDiscriminator`
- 条件输入改为 `10 层 AFM + 12 通道结构图`

#### 4.3 训练损失

第一阶段采用：

- 对抗损失：保证输出更像真实结构图
- 原子图 L1 损失：强约束原子位置
- 键图 L1 损失：强约束局部连接
- 类型图 L1 损失：强约束原子类型

损失权重先采用保守方案：

- 原子图：20
- 键图：10
- 类型图：20

### 5. V19 第一阶段完成标准

在快速 debug 训练里，至少应看到这些趋势：

- `atom_xy_mae` 明显下降
- `type_map_mae` 明显下降
- `typed_center_match` 明显上升
- `atom_type_macro_f1_2d` 不再接近 0
- `ch_collapse_rate_2d` 明显下降

如果这些都不起作用，则说明：

- 不是 3D 主线太难，而是 AFM 到局部化学图本身就没有学起来

### 6. V19 第一阶段对应代码文件

- `src/utils/mol2d.py`
  - 负责从真实坐标和原子类型渲染 2D 结构目标图

- `src/train_v19_stage1.py`
  - 负责训练 AFM -> 2D 分子图的第一阶段模型

- `src/models/baselines.py`
  - 继续复用 `UNetGenerator` 和 `PatchDiscriminator`

- `config_v19_stage1_2d_debug.json`
  - 第一阶段快速调试训练配置

### 7. V19 第二阶段预留接口

第二阶段暂不实现，但第一阶段的输出形式必须为后续补 Z 做准备。

后续会在 2D 输出基础上增加：

- 原子中心列表
- 每个中心的类型
- 键连接列表
- 环骨架位点
- 侧链连接位点

这些都将作为 Z 轴和 3D 细化阶段的输入。

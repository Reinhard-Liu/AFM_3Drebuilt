## 五、可视化核心观察（修正版）

> 以下结论基于修正后的可视化（Predicted 使用 pred_mask），替代旧版分析。
> 正式图位于 `experiments/v16c_best_eval_fixed/visualizations/` 和 `visual_compar/`。

### 确认的改善
1. **结构不再坍缩**：V16b 的"所有原子挤成一团"问题已完全消除。这是 V16c 最明确的改善。
2. **原子数预测接近 GT**：多数样本 pred/GT 原子数差距在 1-3 个以内（Count ≈ 0.94-1.00）。

### 关键缺陷（修正后确认）

1. **3D 拓扑未被学会**：预测结构只是"正确数量的原子在正确的大致位置"，无法看出 GT 中的环形骨架、分支结构、平面芳香体系。RMSD 在 0.2-0.5 范围不等于"结构相似"——RMSD 反映匈牙利匹配后的平均距离，但分子的 3D 拓扑与 GT 仍有根本差距。

2. **Z 轴压缩严重**：几乎所有样本中，GT 有清晰的 Z 方向展开（上下层），Pred 的 Z 分布被压到接近平面。直接导致 Bottom Recall = 6%。

3. **原子间距偏小**：预测的原子聚集在比 GT 更紧凑的区域内。典型案例：#0（GT 展开 XY ±0.5，Pred ±0.3），#713（GT Z ±0.5，Pred Z ±0.2）。

4. **杂原子类型预测失败**：GT 图有多种颜色（C 黑、H 白、N 蓝、O 红、S 黄、Cl 绿），Pred 几乎全是灰白色小圆点。Type Match = 0.37-0.40 主要由 C/H 高占比（占分子 80%+）驱动，**N/O/S/F/Cl 等杂原子几乎不被预测到**。Type Acc 数值不能理解为"类型已学会"。

5. **5-mol 对比显示差距明显**：Pred 结构视觉上远不如 Top-1/2/3 retrieval 结果（后者是数据库真实分子，有正确颜色和拓扑）。GT Rank 通常在 60-800+。

## 六、V16c 最终判断

### 已解决的关键问题
1. **采样器公式错误** → 已修复（DDIM/DDPM 标准公式 + node mask + CoM removal）
2. **Bond enforcement 死逻辑** → 已修复（使用 IDEAL_BOND_LENGTHS 作为目标，梯度双向生效）
3. **Bond 标准不一致** → 已统一（constraints.py 作为 single source of truth）
4. **Bond Validity = 0** → **已突破**（从恒定 0 提升到 0.27-0.40）

### 数值提升（需谨慎解读）

| 指标 | V16b (debug2 E6) | V16c (best E12) | 变化 | 可视化验证 |
|------|-------------------|-----------------|------|-----------|
| Bond(gt_mask) | 0.000 | **0.395** | 从 0 突破 | 确认有效 |
| Bond(pred_mask) | 0.000 | **0.398** | 从 0 突破 | 确认有效 |
| Type Match | 0.095 | **0.400** | +320% | **过于乐观**：主要由 C/H 驱动，杂原子仍失败 |
| Count Accuracy | 0.316 | **0.309** | 持平 | 与可视化一致 |
| Composite | 0.465 | **0.513** | +10% | 部分由 Type 虚高贡献 |

> 注：V16b 使用旧 Bond 口径（tolerance=30%），V16c 使用新口径（tolerance=25%），
> V16c 的 Bond=0.40 在更严格标准下取得。
> 但 Type Match 的提升被可视化证实为 C/H 偏倚，不代表真正学会了原子类型分辨。

### 仍然存在的核心问题
1. **3D 拓扑未被学会**：预测结构无法还原 GT 的环形/分支几何，只是模糊的原子云
2. **杂原子类型预测失败**：N/O/S/F/Cl 几乎不出现在预测中
3. **Z 轴压缩严重**：Bottom Recall 仅 6%
4. **Count Accuracy 停滞**（~0.26-0.31）
5. **C_pred_guided RMSD 劣化**：guidance 把 RMSD 从 0.19（A/B）推高到 0.35
6. **Epoch 12 是峰值，后续回落**：E13-E15 的 Bond 和 Type 均下降
7. **predicted_rings 仍未接通**：C 配置不是完整 mainline

## 七、为什么 best_gen.pt（E12）优于 E15

| 对比 | E12 | E15 | 分析 |
|------|-----|-----|------|
| Bond(gt) | **0.395** | 0.282 | E12 峰值后回落 |
| Type | **0.400** | 0.347 | 同步回落 |
| Composite | **0.513** | 0.497 | 差 3% |
| RMSD | 0.360 | 0.354 | 几乎持平 |

**原因分析**：
1. **S3 constraint_weight=0.5 可能过强**：E12 达到 bond/type 的最佳平衡点后，继续施加 0.5 权重的约束反而干扰了坐标学习
2. **bond_angle_loss 主导 constraint**：bl=0.001 vs ba=0.624，bond_angle 占约束总量的 ~83%，bond_length 几乎没有学习信号
3. **训练总 epoch 不够**：15 epoch 中只有 E10-E15（6 个 epoch）启用了约束，模型还在剧烈调整期

## 八、V16d 实验建议

### 继续保留的 V16c 正确设计
- 修正后的 DDIM/DDPM 采样器
- Node mask + CoM removal
- 统一的 IDEAL_BOND_LENGTHS / MAX_BOND_DIST
- 修正后的 `_apply_physics_guidance` bond gradient
- 双 bond metrics（gt_mask / pred_mask）
- 三组 sanity eval (A/B/C_pred_guided)
- best_gen.pt 按 composite score 选择
- 修正后的可视化脚本（pred_mask 独立于 GT mask）

### 下一轮最优先要改的 5 个点

**1. 杂原子类型预测是当前最大瓶颈之一**
- V16c 的 type head 几乎只输出 C/H，N/O/S/F/Cl 预测极弱
- 建议：增加 type loss 中杂原子的权重（inverse frequency weighting 或 focal loss）
- 或在训练中增加杂原子分子的采样比例
- 理由：可视化明确证实了这是最严重的失败模式

**2. bond_length_loss 需要独立加权提升**
- 当前 bond_length 在 constraint 内部权重 1.0，但实际值只有 0.001，远低于 bond_angle（0.624）
- 建议：在总 loss 中额外单独加一项 `bond_length_weight * bond_length_loss`（建议 1.0-2.0）
- 或调整内部权重为 `5.0*bond + 0.5*angle + 0.3*planarity`
- 理由：bond_length 是 Bond Validity 的直接驱动力，当前几乎被 angle 淹没

**3. 降低 S3 constraint_weight，延长训练周期**
- S2: 0.2, S3: 0.3（当前 0.3/0.5 过强）
- Epochs: 至少 30，最好 45
- 理由：E12 峰值后回落说明约束过早过强地压制了坐标学习

**4. Z 轴展开和 Bottom atom 恢复**
- 当前 bottom_z_loss 只在 S2/S3 激活且权重弱
- 建议：从 S1 后半段（50%）就开始引入 bottom_z_loss
- 可考虑在 coord loss 中对 Z 分量单独加权
- 理由：Z 压缩是可视化证实的普遍现象，Bottom Recall=6%

**5. Guidance 温和化**
- C_pred_guided 的 RMSD（0.35）显著劣于 A/B（0.19），视觉拓扑也更差
- 建议：guidance step size 从 0.005 降到 0.002
- 只在低噪声步骤启用（t < 500）
- 理由：高噪声阶段施加 bond gradient 会干扰去噪，导致结构更紧凑

### 先不要做的事
- 不做 bond_head
- 不做 CFG / classifier-free guidance
- 不做 XY/Z 解耦
- 不做离散扩散类型
- 不修 predicted_rings 的完整 ring snap 系统
- 不做 MMFF94/GFN2-xTB 主评估

### 关键决策建议
1. **S3 强度**：建议降低到 0.3（当前 0.5 导致 E12 后回落）
2. **C_pred_guided 是否继续作为默认推理主线**：是，但需要降低 guidance step size
3. **是否从头训练**：否，建议从 V16c best_gen.pt warm-start
4. **是否把 epochs 改回更长**：是，建议 30-45 epoch
5. **是否需要优先解决杂原子类型问题**：是，这是可视化证实的最大失败模式

## V19.1 方案：AFM 堆栈到 2D 分子图与 Z 轴/3D 的联合训练

### 1. 当前 V19 和 V19.1 的核心区别

当前 V19 的实际落地情况是：

- 已经开始做第一阶段：AFM 堆栈 -> 2D 分子图
- 目前实现的是一个最小 2D 训练脚本
- 训练目标包括：
  - 原子占据图
  - 键图
  - 10 类原子类型图
- 但第二阶段“2D 图 + AFM 堆栈 -> Z 轴与 3D 几何”还只是规划，没有进入训练流程

也就是说，当前 V19 还是“先做 2D，后面再做 3D”的顺序式方案。

V19.1 的目标是把这条路线改成：

- 不是先训练一个 2D 模型，再训练一个 3D 模型
- 而是在一个联合模型里同时训练：
  - 2D 原子/键/类型
  - Z 轴信息
  - 由 2D 与 Z 共同组成的 3D 结构初值

一句话区别：

- V19：分阶段、顺序式
- V19.1：共享编码器、联合训练式

### 2. 为什么 V19 需要升级成 V19.1

如果仍然把 2D 和 3D 完全拆开，会有两个问题：

1. 第一阶段学到的 2D 结构，不一定真的被第二阶段充分利用
2. 第二阶段的 3D 目标不能反向逼第一阶段学更适合 3D 的 2D 结构

V19.1 的联合训练要解决的就是：

- 让 2D 分子图分支直接为 3D 重建服务
- 让 3D 重建的误差信号也反向推动 2D 分支学到更有用的结构表示

### 3. V19.1 的联合模型总结构

V19.1 采用共享一个 AFM 编码主干、同时分出 2D 和 3D 分支的设计。

整体流程：

1. AFM 10 层图像堆栈输入共享编码器
2. 共享特征进入 2D 分子图分支
3. 2D 分支输出的结构图，再和 AFM 特征一起进入 Z 轴/3D 分支
4. Z 轴/3D 分支输出 3D 结构初值
5. 后续再保留轻量几何细化能力

### 4. V19.1 在项目中的具体设计

#### 4.1 共享编码器

项目中的具体实现：

- 不从零重写 AFM 编码器
- 先复用现有适合图像到图像任务的 2D U-Net 主体
- 输入仍然是 10 层 AFM 堆栈
- 编码后的中间特征同时供 2D 分支和 Z 分支使用

这样做的原因：

- 当前最需要先验证的是“联合训练是否比只做 2D 更合理”
- 因此优先做最小可运行实现，不先把模型复杂化到 VideoViT + 新 decoder 全部重写

#### 4.2 2D 分子图分支

2D 分支输出结构化通道图，不输出简单 RGB 图片。

最小版本输出 13 个通道：

- 第 1 通道：原子占据图
- 第 2 通道：键图
- 第 3-12 通道：10 类原子类型图
- 第 13 通道：Z 轴图

其中前 12 个通道与 V19 第一阶段一致，第 13 个通道是 V19.1 新增。

#### 4.3 Z 轴图分支

Z 轴图的设计：

- 在每个真实原子中心附近，用一个圆盘区域写入该原子的归一化 Z 值
- 对应通道叫：
  - `z_map`

这样做的含义是：

- 第一阶段先学“这里有没有原子、是什么原子”
- 联合训练时再学“这个位置大概在 3D 空间的什么高度”

这是一种比“直接回归整团 3D 点云”更容易学的中间监督。

#### 4.4 从 2D + Z 恢复 3D 初值

项目中的具体恢复方式：

- 对真实原子中心位置或局部匹配到的预测中心，读取：
  - `xy` 位置
  - `z_map` 对应高度
- 形成一个最小 3D 初值

这个初值还不是最终分子几何，但可以用于验证两个关键问题：

1. 联合训练是否让 Z 信息也开始被学到
2. 2D 结构质量是否能正向带动 3D 初值质量

#### 4.5 后续 3D 细化接口

V19.1 第一批实现不直接重构完整 3D 扩散主线，但要保留接口：

- 后续 3D 细化器的输入应是：
  - 2D 原子中心
  - 原子类型
  - 键连接
  - z_map
  - AFM 编码特征

这样未来能自然接入：

- 轻量几何修正网络
- 或者把现有扩散模型改成“细化器”而不是“从零发明结构的主生成器”

### 5. V19.1 的训练损失设计

#### 5.1 2D 结构损失

- 原子图 L1 损失
- 键图 L1 损失
- 类型图 L1 损失
- 对抗损失

#### 5.2 Z 轴损失

- `z_map_l1_loss`
  中文：Z 轴图平均绝对误差
  含义：预测的 Z 图和真实 Z 图的差距

#### 5.3 3D 初值损失

最小版本不直接对整团 3D 点云做完整强监督，而是先用软监督：

- `atom_z_mae_r3`
  中文：原子局部 Z 偏差
  含义：在真实原子中心半径 3 像素内，读取预测 Z 后与真实 Z 比较

这样避免一开始把任务又拉回“整团 3D 点云直接回归”。

### 6. V19.1 的训练指标

V19.1 训练过程主要看连续值指标，不主用通过率。

主要指标：

- `atom_xy_mae`
  中文：二维原子位置平均偏差

- `bond_map_mae`
  中文：二维键图平均偏差

- `type_map_mae`
  中文：二维原子类型图平均偏差

- `atom_center_score_r3`
  中文：原子中心软匹配分数

- `typed_center_score_r3`
  中文：位置和类型同时正确的软匹配分数

- `type_top1_local_acc_r3`
  中文：局部原子类型准确率

- `atom_type_macro_f1_2d`
  中文：二维原子类型宏平均 F1

- `ch_collapse_rate_2d`
  中文：二维碳氢塌缩率

- `z_map_mae`
  中文：Z 轴图平均偏差

- `atom_z_mae_r3`
  中文：原子局部 Z 偏差

### 8. V19.1 第一批实现的最小目标

第一批不追求完整 3D 分子完全恢复，只验证联合训练猜想是否成立。

最小目标：

1. 联合模型能稳定前向和训练
2. 2D 指标不因加入 Z 分支而明显变差
3. `z_map_mae` 能正常下降
4. `atom_z_mae_r3` 能正常下降

只要这四点成立，就说明：

- 2D 图和 Z 轴可以在一个模型里联合学习
- 这条路线值得继续扩展到真正的 3D 细化阶段

### 9. V19.1 当前判断标准

如果联合训练后：

- `atom_xy_mae` 持平或更好
- `typed_center_score_r3` 持平或更好
- `type_top1_local_acc_r3` 不下降
- 同时 `z_map_mae` 明显下降

则说明：

- 联合训练设计合理
- 当前路线比“先全做 2D、再完全分开做 3D”更值得继续

如果联合训练明显拖垮 2D 指标，则说明：

- 当前共享方式太强或太早
- 需要进一步改成“共享编码器 + 分阶段损失权重调度”

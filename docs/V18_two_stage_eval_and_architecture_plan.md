# V18 两阶段方案：原子级结构忠实重建优先

## 一、为什么要重置 V18

这次重新设计 V18，不是因为前面的方向完全错了，而是因为第一阶段的目标被理解偏了。

之前那版 V18 把“视觉重建”偏向理解成：

- 整体轮廓像不像
- 全局形状像不像
- 三维体积重叠高不高

这些内容当然有价值，但它们更接近“外形相似”，不是你真正要的“结构忠实重建”。

你现在明确提出的第一阶段目标是：

**模型根据 AFM 图像重建出的三维坐标结构，必须尽量还原同一个分子的原子级结构，而不只是画出一个差不多的轮廓。**

这意味着第一阶段真正关心的是：

- 原子总数必须一致
- 对应原子的位置偏差不能大
- 对应原子类型要尽量准确
- 环结构要保持完整
- 化学键和局部几何要合理

因此，V18 第一阶段必须从“轮廓相似优先”，改成：

**原子级结构忠实重建优先。**

第二阶段才是：

**在第一阶段的高质量重建基础上，把测试时的真实编号命中率做高。**

## 二、V18 的两个最终目标

### 第一阶段目标

**把三维坐标结构重建得像真实分子本身。**

这里的“像”不是看起来大概像，而是指：

- 原子数量对
- 对应原子的位置对
- 对应原子类型对
- 环结构完整
- 连接关系合理
- 局部化学几何不离谱

### 第二阶段目标

**用重建出的三维坐标结构去检索测试集时，真实分子的编号尽量稳定进入前 5。**

因此，V18 必须明确拆成两阶段：

- 第一阶段：先把结构本身重建对
- 第二阶段：再把检索命中率做高

## 三、V18 评估设计总原则

### 3.1 第一阶段不再让旧综合分数主导

从 V18 开始，旧的“综合分数”不再作为第一阶段主排序指标。

原因很直接：

- 旧分数过于平均
- 它会把一些真正关键的问题稀释掉
- 例如原子类型错很多、杂原子放错地方、环断裂、原子数不一致，旧分数都可能显得没有那么严重

而你要的第一阶段，恰恰就是要把这些问题拉成主矛盾。

### 3.2 第一阶段要分成“训练初期软指标”和“最终严格指标”

训练刚开始时，不应该直接用最严格的通过率和严格连接边指标来判断方向是否正确。

原因：

- 原子总数、严格连接边、严格环完整率在训练初期几乎一定偏低
- 如果一开始就用这些硬指标做主判断，很多本来在变好的方向也会被误判成“完全没效果”

因此，V18 第一阶段要明确拆成两层：

- 训练初期：
  主要看“容差命中率”一类软指标  
  例如：有多少重原子已经摆到正确位置附近、有多少杂原子已经出现在大致正确的位置、有多少侧链已经接到正确骨架位点附近
- 最终评估：
  再看严格通过率、严格连接边 F1、严格环完整率

也就是说：

- `struct_fidelity_pass_rate` 和 `attachment_edge_f1` 仍然保留
- 但它们主要用于最终报告，不再用于训练初期判断方向

### 3.3 第一阶段仍然优先看“原子级一致性”

第一阶段的主评价顺序必须是：

1. 原子总数是否一致
2. 对应原子位置是否准确
3. 对应原子类型是否准确
4. 环结构是否完整
5. 化学键和局部几何是否合理
6. 整体形状是否也基本一致

注意最后一条仍然保留，但它不再是第一阶段的主导项，而是辅助项。

### 3.3 第二阶段才把编号命中率提到主位

第二阶段的重点是：

- 前 1 命中率
- 前 5 命中率
- 平均倒数排名

但第二阶段必须建立在第一阶段已经有一定通过率的基础上。

如果第一阶段连同一分子的原子级结构都重建不出来，那么第二阶段即使检索偶尔命中，也很难稳定。

## 四、统一字段命名规则

从 V18 开始，日志、可视化标题、JSON 文件、实验报告、横向对比表都必须统一字段名。

统一规则如下：

- 单样本或单次评估字段：直接使用基础字段名  
  例如：`matched_atom_rmsd`
- 均值字段：基础字段名加 `_mean`  
  例如：`matched_atom_rmsd_mean`
- 标准差字段：基础字段名加 `_std`
- 布尔通过字段：基础字段名加 `_pass`
- 分组评分字段：分组名加 `_score`
- 命中率字段：统一使用 `hit_at_k`

从这版 V18 开始，下列旧字段不再作为第一阶段主排序键：

- `composite_score`
- `visual_pass_rate`
- `visual_recon_score`

它们最多只保留为历史兼容字段或辅助对照字段。

## 五、第一阶段评估体系：原子级结构忠实重建

## 5.1 第一阶段主指标

### `struct_fidelity_pass_rate`

中文含义：

**原子级结构忠实重建通过率**

定义：

一个样本只有同时满足下面几项，才算“通过”：

- `atom_count_exact = 1`
- `matched_heavy_atom_rmsd` 达标
- `atom_type_acc` 达标
- `heteroatom_f1` 达标
- `ring_complete_rate` 达标
- `attachment_edge_f1` 达标
- `bond_validity_pred` 达标

最终统计通过样本比例，得到 `struct_fidelity_pass_rate`。

这项指标保留，但主要用于最终评估和最终模型对比，不再用于训练初期判断方向。

### `struct_fidelity_score`

中文含义：

**原子级结构忠实重建总分**

这个总分不再强调轮廓优先，而强调结构忠实度优先。

建议公式如下：

```text
struct_fidelity_score =
    0.15 * atom_count_score
  + 0.25 * atom_position_score
  + 0.20 * atom_semantic_score
  + 0.15 * ring_integrity_score
  + 0.10 * connectivity_score
  + 0.10 * local_chem_score
  + 0.05 * global_shape_aux_score
```

这里的意思很明确：

- 原子位置和原子类型是主轴
- 原子数和环结构是硬要求
- 连接关系和局部化学不能掉
- 整体轮廓仍然看，但只做辅助

## 5.2 第一阶段字段设计

### A0. 训练初期软指标

这一组字段专门回答：

**模型有没有开始把原子摆到对的位置附近，而不是一步到位完全正确。**

#### `heavy_atom_hit_rate_tight`

中文含义：

**重原子严格容差命中率**

说明：

- 真实重原子中，有多少能在较严格的容差范围内找到预测原子
- 允许细微坐标偏差
- 不要求坐标完全一致

#### `heavy_atom_hit_rate_medium`

中文含义：

**重原子中等容差命中率**

说明：

- 这是训练初期最重要的方向指标之一
- 直接反映“有多少主体原子已经摆到了正确位置附近”

#### `heavy_atom_hit_rate_loose`

中文含义：

**重原子宽松容差命中率**

说明：

- 用来判断模型是否至少已经把原子摆到大致正确区域

#### `typed_heavy_atom_hit_rate_medium`

中文含义：

**位置和类型都正确的重原子中等容差命中率**

说明：

- 不只要求位置接近
- 还要求原子类型也正确
- 比单纯位置命中率更接近真正的重建能力

#### `heteroatom_hit_rate_medium`

中文含义：

**杂原子中等容差命中率**

说明：

- N、O、S、卤素等关键原子中，有多少在容忍范围内被正确放到对应位置

#### `ring_site_coverage_medium`

中文含义：

**环位点中等容差覆盖率**

说明：

- 真实环骨架位点中，有多少被预测原子覆盖到
- 不要求严格恢复完整环边，只看骨架位点有没有开始成形

#### `soft_attachment_site_acc_medium`

中文含义：

**软连接位点正确率**

说明：

- 只要求侧链接到了正确骨架位点附近
- 允许连接位置存在小偏差
- 不要求严格边完全一模一样

#### `macro_type_f1`

中文含义：

**原子类型宏平均 F1**

说明：

- 每类原子单独计算，再平均
- 用来防止模型把几乎所有原子都塌成碳和氢时，平均类型准确率仍然看起来不低

#### `ch_collapse_rate`

中文含义：

**碳氢塌缩率**

说明：

- 用来衡量模型是否过度把原子预测成碳/氢主类
- 越高说明类型塌缩越严重

#### `soft_recon_score`

中文含义：

**训练初期软重建总分**

说明：

- 这是训练初期用于判断方向是否正确的主分数
- 它优先看容差命中率、杂原子命中、软连接位点、环位点覆盖，而不是严格通过率

### A. 原子数量组

这一组回答：

**模型到底有没有把分子里的原子总数重建对。**

#### `atom_count_exact`

中文含义：

**原子总数是否完全正确**

取值：

- `1` 表示预测原子总数和真实完全相同
- `0` 表示不相同

#### `atom_count_abs_error`

中文含义：

**原子总数绝对误差**

说明：

- 预测原子数和真实原子数相差多少
- 越小越好

#### `atom_count_score`

中文含义：

**原子数量评分**

建议由以下两项综合：

- `atom_count_exact`
- `atom_count_abs_error`

这个评分要明确体现：完全正确比“差 1 个”更重要。

### B. 原子位置组

这一组回答：

**对应原子的位置有没有真正对上。**

#### `matched_atom_rmsd`

中文含义：

**对应原子匹配后的整体三维位置误差**

说明：

- 先给预测原子和真实原子建立对应关系
- 再计算匹配后的三维位置误差
- 这是第一阶段最核心的几何指标之一

#### `matched_heavy_atom_rmsd`

中文含义：

**对应非氢原子匹配后的三维位置误差**

说明：

- 非氢原子更决定骨架和官能团位置
- 这项比全原子位置误差更能反映结构主体是否对上

#### `matched_atom_mae`

中文含义：

**对应原子平均绝对位置误差**

说明：

- 反映平均每个对应原子偏了多少
- 和均方误差一起用，更稳定

#### `atom_position_score`

中文含义：

**原子位置评分**

建议由以下几项综合：

- `matched_atom_rmsd`
- `matched_heavy_atom_rmsd`
- `matched_atom_mae`

### C. 原子语义组

这一组回答：

**原子类型是否对，尤其是关键杂原子是否放在了正确位置。**

#### `atom_type_acc`

中文含义：

**对应原子类型准确率**

说明：

- 不是简单看整体类别分布
- 而是看匹配后的对应原子类型是否一致

#### `heteroatom_precision`

中文含义：

**杂原子预测精确率**

说明：

- 预测成杂原子的那些位置，有多少是真的杂原子

#### `heteroatom_recall`

中文含义：

**杂原子召回率**

说明：

- 真实杂原子里，有多少被模型找了回来

#### `heteroatom_f1`

中文含义：

**杂原子综合准确度**

说明：

- 同时考虑精确率和召回率
- 第一阶段必须把它拉高，因为这直接关系到“是不是同一个分子”

#### `atom_semantic_score`

中文含义：

**原子语义评分**

建议由以下几项综合：

- `atom_type_acc`
- `heteroatom_precision`
- `heteroatom_recall`
- `heteroatom_f1`

### D. 环结构完整性组

这一组回答：

**真实分子里的环结构，模型到底保住了多少，而且是否保得正确。**

#### `ring_count_exact`

中文含义：

**环系数量是否完全正确**

说明：

- 这里不再只看单个环
- 更强调环系数量是否对

#### `ring_complete_rate`

中文含义：

**真实环系完整保留比例**

说明：

- 真实分子里的环系，有多少在预测结构里完整保留下来
- 要求不是“有个大概的环”，而是环没有断、没有塌、没有明显丢原子

#### `ring_size_acc`

中文含义：

**环大小准确率**

说明：

- 五元环、六元环等是否判断正确

#### `ring_aromatic_acc`

中文含义：

**芳香性判断准确率**

说明：

- 芳香环是否仍然表现出正确的环内化学特征

#### `ring_fusion_f1`

中文含义：

**环系融合关系综合准确度**

说明：

- 关注两个环是不是正确共享边、共享位点或形成正确融合关系

#### `ring_integrity_score`

中文含义：

**环结构完整性评分**

建议由以下几项综合：

- `ring_count_exact`
- `ring_complete_rate`
- `ring_size_acc`
- `ring_aromatic_acc`
- `ring_fusion_f1`

### E. 连接关系组

这一组回答：

**侧链和骨架是不是接对了位置。**

#### `attachment_edge_f1`

中文含义：

**骨架与外部连接关系综合准确度**

说明：

- 真实结构里哪些位置连出了侧链
- 这些侧链是否接在了正确的骨架位点

#### `scaffold_relation_f1`

中文含义：

**骨架之间关系综合准确度**

说明：

- 如果有多环系或多骨架系统
- 它们之间的连接、融合、桥接是否正确

#### `connectivity_score`

中文含义：

**连接关系评分**

建议由以下几项综合：

- `attachment_edge_f1`
- `scaffold_relation_f1`

### F. 局部化学几何组

这一组回答：

**结构看起来像了以后，局部化学几何是否也合理。**

#### `bond_validity_pred`

中文含义：

**按预测原子数截取后的化学键合理性**

说明：

- 第一阶段主评估必须优先用这项
- 因为多预测出来的错误原子也会破坏真实重建效果

#### `bond_length_score`

中文含义：

**键长合理性评分**

说明：

- 预测结构里的键长有多接近合理范围

#### `bond_angle_score`

中文含义：

**键角合理性评分**

说明：

- 预测结构里的局部角度是否符合合理几何

#### `ring_planarity_score`

中文含义：

**环平面性评分**

说明：

- 特别用于芳香环和平面杂环
- 检查环内原子是否落在合理平面附近

#### `torsion_tfd`

中文含义：

**扭转相似度偏差**

说明：

- 用来辅助判断整体构象是否接近
- 越低越好

#### `local_chem_score`

中文含义：

**局部化学几何评分**

建议由以下几项综合：

- `bond_validity_pred`
- `bond_length_score`
- `bond_angle_score`
- `ring_planarity_score`
- `torsion_tfd`

### G. 全局形状辅助组

这一组不再决定第一阶段主排序，但必须保留为辅助诊断。

这一组回答：

**整体轮廓有没有明显跑偏。**

#### `symm_rmsd`

中文含义：

**考虑对称等价后的最佳三维对齐误差**

说明：

- 参考 RDKit 的对称对齐思路
- 解决对称原子重新排序后普通位置误差失真的问题

#### `norm_rmsd`

中文含义：

**按分子大小归一化后的三维误差**

说明：

- 防止大分子和小分子直接比较时失真

#### `shape_tanimoto`

中文含义：

**三维体积形状重叠分数**

说明：

- 看两个结构整体体积轮廓有多像
- 越高越好

#### `shape_protrude`

中文含义：

**三维形状突出差异分数**

说明：

- 看预测结构是否多出一块、鼓出一块或者拖出一段
- 越低越好

#### `global_shape_aux_score`

中文含义：

**全局形状辅助评分**

建议由以下几项综合：

- `symm_rmsd`
- `norm_rmsd`
- `shape_tanimoto`
- `shape_protrude`

### H. 遮挡与底部恢复组

这一组继续保留，但只作为辅助诊断，不再给过高权重。

#### `occluded_recovery_score`

中文含义：

**遮挡区域和底部原子恢复评分**

说明：

- AFM 图像里底部和被遮挡区域仍然困难
- 这项保留，是为了知道模型是不是只会画“上半部分”

## 5.3 原子对应关系如何建立

第一阶段要评估“对应原子位置”和“对应原子类型”，因此必须先有稳定的原子对应关系。

V18 统一规定：

1. 优先用图一致的对应关系  
   即优先使用基于图同构、最大公共子结构或稳定骨架位点的对应关系。

2. 对称原子允许等价重排  
   即不能因为对称原子顺序不同，就把结构判错。

3. 只有在图一致匹配失败时，才回退到距离约束下的最优匹配  
   这种回退必须在报告里单独标记。

因此，后续报告里应增加两个辅助字段：

- `atom_match_mode`：原子对应建立方式
- `atom_match_coverage`：成功建立对应的原子覆盖比例

这两个字段不进入主评分，但必须进入诊断报告。

## 5.4 第一阶段模型选择规则

从 V18 开始，第一阶段选最好模型时，排序规则改成：

1. `struct_fidelity_pass_rate`
2. `struct_fidelity_score`
3. `matched_heavy_atom_rmsd`
4. `ring_integrity_score`
5. `bond_validity_pred`

旧的 `composite_score` 只作为历史对照，不再决定第一阶段最好模型。

## 六、第二阶段评估体系：编号命中率优化

第二阶段的前提是：

- 第一阶段已经能把结构重建到可用水平
- `struct_fidelity_pass_rate` 不再长期为 0

第二阶段主字段如下。

### `cid_hit_at_1`

中文含义：

**真实编号命中前 1 的比例**

### `cid_hit_at_5`

中文含义：

**真实编号命中前 5 的比例**

这是第二阶段最重要的主指标。

### `cid_mrr`

中文含义：

**真实编号平均倒数排名**

说明：

- 排名越靠前，这项越高
- 可以更稳定地区分“命中但排得靠后”和“命中且排得很前”

### `recon_fail_rate`

中文含义：

**因为重建质量不够而导致失败的比例**

说明：

- 如果第一阶段结构就不对，检索失败应记在这里

### `ranking_fail_rate`

中文含义：

**结构已经基本对，但排序仍未把真实编号排进前列的失败比例**

说明：

- 如果第一阶段已经基本过关，第二阶段还失败，就说明问题在检索排序而不是重建本身

## 七、和现有项目架构的关系：哪些保留，哪些必须改

## 7.1 当前架构里仍然正确的部分

当前项目里这些方向仍然保留：

- 继续使用显式结构层，而不是退回纯坐标生成
- 继续使用“环系骨架 + 连接位点 + 侧链关系”的主线
- 继续保留骨架 token 条件化，而不是只做生成后修补
- 继续保留局部化学几何约束

这些方向没有问题。

## 7.2 当前架构和新评估体系不一致的地方

### 问题 1：原子总数还没有被当成第一阶段硬要求

当前架构里，原子总数预测仍然更像一个辅助量。

但在新的第一阶段里：

- 原子总数完全一致是通过条件
- 不是“差不多就行”

因此，原子数预测必须升级成更强的主分支，而不是只保留一个总数头。

### 问题 2：结构层还不够稳定地支持“原子对应”

第一阶段现在要看“对应原子位置”和“对应原子类型”，这要求：

- 骨架位点编号更稳定
- 环系顺序更稳定
- 连接位点语义更稳定

而当前实现里，某些环位点编号仍然不够稳定，导致模型和评估都不够稳。

### 问题 3：杂原子定位还没有被拉成主任务

新的第一阶段里，杂原子准确度是主指标之一。

这意味着：

- 杂原子不能再只是类型平均准确率里的一小部分
- 必须单独建模、单独看、单独加权

### 问题 4：局部化学合理性不能再只依赖生成后修补

新的第一阶段要求：

- 环不只是看起来像
- 键也要合理
- 局部几何也要合理

因此，局部化学几何必须在生成过程中更早参与，而不是主要依赖最后再修一遍。

### 问题 5：整体形状分支不能继续当第一阶段主导

整体形状当然要保留，但它只能做辅助。

第一阶段现在真正的主导项应该是：

- 原子数
- 原子位置
- 原子类型
- 环结构
- 连接关系

## 十二、V18 的一句话结论

V18 第一阶段不再追求“轮廓看起来像”，而要追求：

**这是不是同一个分子的原子级三维结构。**

只有第一阶段把这件事做好，第二阶段的真实编号前 5 命中率才有可能稳定地高。

## 十三、当前实测结论与下一步优先级

在这版 V18 第一阶段指标接入后，已经对现有几条主要 checkpoint 做了小规模横评。

当前排序结果说明了两件事：

1. 旧的综合分数并不能代表第一阶段真正的结构忠实度
2. 当前最突出的失败项不是“整体轮廓完全不对”，而是：
   - 杂原子几乎没有真正放对
   - 连接关系几乎没有真正接对
   - 环和连接一旦进入更严格的结构口径，很多版本都过不了

### 13.1 当前阶段观察到的直接结论

1. `struct_fidelity_pass_rate` 仍然接近 0  
   这意味着现在还没有版本真正达到“原子级结构忠实重建通过”的水平。

2. `heteroatom_f1` 很低  
   说明当前模型虽然有时能把整体骨架画得接近，但杂原子位点几乎没有学稳。

3. `attachment_edge_f1` 很低  
   说明侧链和骨架的连接关系还没有真正接对。

4. 某些旧版本在新口径下反而比部分新结构版本更强  
   这说明：
   - 新结构层方向本身不一定错
   - 但当前新结构层主要改善的是局部几何和某些骨架信息
   - 还没有稳定转化成“原子级结构忠实重建”

### 13.2 这对架构意味着什么

接下来 V18 第一阶段的结构调整重点，不应该继续优先压在：

- 纯轮廓相似
- 纯形状分数
- 只强化环中心或环平面

而应该优先压在：

1. **杂原子位点建模**
   - 不只看平均原子类型准确率
   - 要让模型显式学习“哪些骨架位点应该是杂原子”

2. **连接关系建模**
   - 不只看有没有侧链
   - 要看侧链接在了哪个骨架位点

3. **原子角色建模**
   - 区分骨架原子、连接锚点原子、侧链原子
   - 让模型先知道每个原子在结构里扮演什么角色

4. **数量分解建模**
   - 不只预测总原子数
   - 要预测骨架内数量、骨架外数量、连接数量

### 13.3 下一批架构改动优先级

因此，V18 第一阶段后续最优先的代码改动顺序应当是：

1. 在预测头中加入更明确的原子角色和杂原子位点建模
2. 在结构 token 中加入更稳定的连接位点语义
3. 在训练中单独记录并强调：
   - 杂原子位点损失
   - 连接关系损失
   - 原子角色损失
4. 之后再重新比较：
   - `struct_fidelity_pass_rate`
   - `heteroatom_f1`
   - `attachment_edge_f1`

只有当这三项明显上升时，才能说明结构层真的开始服务第一阶段目标了。

## 十四、V18.2 架构改造主线：先解析结构语义，再生成三维坐标

### 14.1 为什么要从 V18.1 进入 V18.2

V18.1 已经回答了一个重要问题：

- “原子角色 / 连接位点 / 杂原子位点” 这类语义监督可以稳定训练
- 但如果这些语义只是训练末端的辅助分类，它们不会自动改变生成结果

也就是说，当前的问题不是“模型完全学不会这些语义”，而是：

- 语义出现得太晚
- 语义太粗
- 语义没有真正进入生成链路

因此，V18.2 不再把这类语义当成“结果出来以后再解释一下”的辅助头，
而是把它们前移为：

**图像先解析出结构语义，再用这些结构语义去条件化三维生成。**

### 14.2 V18.2 的分层架构

V18.2 的推荐主线是三层：

1. **图像到结构语义层**
   - 从 AFM 图像先解析出：
     - 有哪些骨架系统
     - 每个骨架系统里有哪些稳定位置
     - 哪些位置是侧链连接位点
     - 哪些位置是杂原子位点

2. **结构语义到分子连接层**
   - 用上一步的结果表达：
     - 哪些原子属于骨架
     - 侧链应该接到哪个骨架位置
     - 杂原子应该落在哪个骨架位置
     - 哪些非骨架原子属于同一条侧链

3. **分子连接到三维几何层**
   - 再把这些结构语义输入去噪生成器
   - 让模型不是盲目生成一团原子，再靠后处理修补
   - 而是在生成时就知道：
     - 哪些位置是骨架核心
     - 哪些位置要接侧链
     - 哪些位置应该放杂原子

### 14.3 V18.2 不再沿用的旧思路

从这一版开始，以下做法不再作为第一阶段主线：

- 只靠“整体轮廓”或“全局形状”去推结构对不对
- 先直接生成全部原子坐标，再靠晚期辅助头解释
- 只做“是不是连接锚点 / 是不是杂原子”的二值标签
- 只优化旧综合分数

这些做法可以保留为辅助信息，但不能再决定架构方向。

### 14.4 V18.2 第一批必须补齐的结构语义标签

这批标签的目标很明确：

**让模型知道“哪个具体骨架位置应该承担什么结构角色”。**

建议统一使用下面这些字段名：

1. `scaffold_atom_canonical_site_index`
   - 中文含义：骨架原子的稳定位点编号
   - 表示某个骨架原子在所属骨架系统中的稳定位置编号
   - 这是后续连接位点和杂原子位点的基准坐标系

2. `scaffold_atom_attachment_target_site`
   - 中文含义：原子所归属的骨架连接位点编号
   - 对骨架连接原子来说，就是它自己的位点编号
   - 对侧链原子来说，是它最终挂接到的那个骨架位点编号

3. `scaffold_atom_hetero_target_class`
   - 中文含义：骨架位点的元素类别目标
   - 不再只是“是不是杂原子”
   - 而是明确这个位点应该是什么元素类别

4. `scaffold_atom_sidechain_root_site`
   - 中文含义：侧链原子的入口骨架位点编号
   - 用于把同一条侧链稳定归到某个骨架位点之下

5. `scaffold_system_site_count`
   - 中文含义：每个骨架系统包含多少个稳定位点
   - 用于后续的结构 token 组织和条件化

### 14.5 V18.2 第一批结构 token

V18.2 的最小结构 token 不再只包含：

- 骨架系统 token
- 骨架关系 token
- 连接原子 token
- 侧链边 token

还要补进：

1. **骨架位点 token**
   - 表达每个骨架系统内部的具体稳定位点
   - 包含位点坐标、位点编号、所属骨架系统、位点元素类别、是否为连接位点

2. **骨架位点元素 token**
   - 重点表达哪些位点应该放杂原子
   - 让模型在生成前就看到“这个骨架位置的元素语义”

这一步的目标不是立刻做完整预测式解析器，
而是先做：

**真实结构语义条件化实验。**

### 14.6 V18.2 的验证顺序

V18.2 不能一上来就做长时间闭环训练，必须按下面顺序验证：

1. **真实结构语义条件化 debug**
   - 把真实的骨架位点、连接位点、元素位点 token 直接喂给生成器
   - 先验证：
     - `heteroatom_f1`
     - `attachment_edge_f1`
     - `ring_complete_rate`
     能不能明显上升

2. **如果真实结构语义条件化有效**
   - 再去做预测式结构解析器
   - 也就是把图像特征先变成这些结构语义 token

3. **如果真实结构语义条件化无效**
   - 说明不是预测器不够强
   - 而是当前结构语义设计还不够对
   - 此时应先改标签和 token 设计，而不是继续训练更久

### 14.7 V18.2 的当前实施顺序

本轮实施顺序固定为：

1. 先把 V18.2 架构思路和字段写入方案文档
2. 补位点级标签
3. 把真实结构语义做成 token
4. 用真实结构语义条件化生成器做小规模 debug
5. 再决定是否进入预测式结构解析器

### 14.8 这版 V18.2 要回答的关键问题

这一版最核心的问题只有一个：

**如果把正确的骨架位点、连接位点、元素位点语义提前告诉生成器，它能不能明显提高原子级结构忠实重建。**

只有这个问题得到肯定答案，后面的预测式结构解析器才值得继续做。

## 十五、V18.3 最小实施路线：位点语义从软提示升级为槽位硬条件

V18.2 的实验已经说明：

- 真实位点语义 token 作为全局交叉注意力提示，仍然太弱
- 它可以让模型“看到”正确语义，但还不足以让每个原子槽位“按这个语义去生成”

因此，V18.3 的最小实施路线不再继续堆更多晚期辅助头，而是直接做三件事：

1. **补位点对象标签**
   - 把骨架位点编号、连接目标位点、侧链入口位点、位点元素类别补成正式字段

2. **增加槽位语义监督**
   - 不只预测“是不是连接位点 / 是不是杂原子”
   - 还要预测：
     - 属于哪个位点
     - 接到哪个位点
     - 属于哪条侧链入口
     - 该位点应该是什么元素类别

3. **做真实位点槽位硬条件化**
   - 让每个原子槽位直接接收自身的结构语义偏置
   - 而不是只共享一组全局结构 token

这一版的目标不是最终闭环，而是先回答：

**如果每个原子槽位都拿到正确的结构语义，模型是否会明显改善原子级结构忠实重建。**

## 十六、V18.4 最小实施路线：真实连接图硬条件化

V18.3 的结果已经说明：

- 槽位语义硬条件比全局 token 更有效
- 但当前最卡的指标仍然是 `attachment_edge_f1`
- 也就是说，模型现在不是“不知道哪些槽位重要”，而是“还没有真正按正确的连接关系去生成”

因此，V18.4 的重点不再是继续补更多语义标签，而是直接把**真实连接图**接进生成器。

### 16.1 V18.4 的核心思路

把真实结构中的两类连接直接作为槽位级硬条件：

1. **骨架内部局部边**
   - 表达骨架原子之间哪些位置本来就相连

2. **骨架到侧链、以及侧链内部局部边**
   - 表达哪些槽位应该围绕同一条连接链展开

这一步不是只把边做成 token 给交叉注意力参考，
而是直接把它做成**槽位连接图条件**，送到去噪器层内做邻接消息注入。

### 16.2 V18.4 要验证的问题

这一版只回答一个问题：

**如果把正确的槽位连接图直接喂给生成器，骨架到侧链的连接关系能不能明显恢复。**

重点观察的字段就是：

- `attachment_edge_f1`
- `ring_complete_rate`
- `heteroatom_f1`
- `struct_fidelity_pass_rate`

### 16.3 V18.4 的最小实现范围

这一版先不做预测式连接图解析器，只做真实连接图上限实验。

具体做法：

1. 从现有骨架标签里直接构建槽位连接图
2. 区分两种边：
   - 骨架局部边
   - 侧链相关边
3. 在去噪器每层前，把当前槽位特征按这张图做一次邻接消息聚合
4. 用这个版本做小规模 debug

### 16.4 V18.4 成功的判断标准

如果这一版有效，应至少看到：

- `attachment_edge_f1` 不再恒定为 `0`
- `ring_complete_rate` 明显上升
- `struct_fidelity_score` 同时改善，而不是只换来更低的位置误差

如果这一步仍然拉不动连接边指标，
就说明下一步不能只做“更强条件化”，而要进入真正的**显式连接图预测器**。

## 十七、V18.5 最小实施路线：显式位点图解析器

V18.4 的结果进一步说明：

- 只把真实连接图作为层内硬条件送进生成器，仍然不足以让模型真正学会“谁接谁”
- 当前主干仍然是“先生成原子，再由辅助头解释结构”
- 这和 V18 第一阶段新的软指标并不对齐

因此，V18.5 的重点不再是继续给当前原子生成器补更多晚期监督，
而是增加一个更靠前的**位点图解析器**。

### 17.1 V18.5 的核心思路

先从 AFM 图像的 patch 特征中直接解析一组“骨架位点对象”，再预测这些位点之间的局部连接图。

这里的“位点对象”不是完整原子集合，而是更稳定的结构锚点，至少包含：

- 这个位点是否存在
- 这个位点大致在哪个空间位置
- 它属于哪个骨架系统
- 它更像什么元素类别
- 它是不是侧链连接位点
- 它与哪些位点相连

这样做的目的，是把结构层前移，让模型先学会“分子大概由哪些关键位点组成”，
再去决定完整原子应该怎么围绕这些位点展开。

### 17.2 V18.5 要验证的问题

这一版主要验证三件事：

1. 从 AFM 图像里，模型能不能先把骨架位点对象本身解析出来
2. 位点对象之间的局部连接关系能不能先学出来
3. 如果这层先学稳，后续是否更有机会把下列软指标真正拉起来：
   - `heavy_atom_hit_rate_medium`
   - `typed_heavy_atom_hit_rate_medium`
   - `heteroatom_hit_rate_medium`
   - `ring_site_coverage_medium`
   - `soft_attachment_site_acc_medium`

### 17.3 V18.5 的最小实现范围

第一批 V18.5 不直接闭环进生成器，只做对象级解析训练入口。

具体做法：

1. 新增 `SiteGraphParserHead`
   - 从 AFM patch 特征中输出固定数量的位点 query
   - 每个 query 预测：
     - 位点是否存在
     - 位点坐标
     - 所属骨架系统
     - 位点元素类别
     - 是否为连接位点
     - 是否为杂原子位点

2. 同时预测位点对之间的局部连接
   - 先只覆盖骨架内部局部连接
   - 不急着一步做到完整的侧链连接图

3. 训练上先作为独立辅助分支接入
   - 目标不是立刻替代现有生成器
   - 而是先回答“图像里有没有足够的位点级结构信息可被解析出来”

### 17.4 V18.5 成功的判断标准

如果这一步有效，应至少看到：

- 位点对象有效性损失下降
- 位点坐标损失下降
- 位点局部边损失下降
- 后续再接闭环时，更有机会带动：
  - `ring_site_coverage_medium`
  - `soft_attachment_site_acc_medium`
  - `typed_heavy_atom_hit_rate_medium`

如果连这一步都学不动，
就说明问题更早，主干编码器本身还提不出足够稳定的结构对象信息。

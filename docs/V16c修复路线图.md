# V16c 修复路线图

## 1. V16c 与 V16b 的关系

V16c 不是 V16b 的调参迭代，而是对 V16b 暴露的**结构性失效**进行根因修复。

V16b 的训练 loss 正常下降，但端到端生成完全坍缩（所有预测原子挤成一团）。经过系统分析，确认问题不在超参数或训练轮数，而在于：
- 反向扩散采样器公式写错，数学上是恒等式
- 原子数预测没有真正约束生成过程
- 训练与推理的 mask 口径不一致

V16c 的目标是：**先恢复正确的生成主链，再考虑任何增强。**

所有 V16c 产物写入 `experiments/v16c/`，不覆盖 V16b。

## 2. 保留 V15 的核心设计

以下设计必须保留，不允许回退：

| 设计 | 保留原因 |
|------|---------|
| **固定坐标系** | AFM 图像在固定扫描坐标系下采集，输入-输出是配准关系，不是自由旋转对称任务。GeoDiff/EDM 的 SE(3) 等变性设计用于无条件自由分子生成，不适用于本任务 |
| **不做旋转增强** | 同上，AFM 图像和分子坐标已对齐，旋转增强会破坏配准关系 |
| **coord_head 直接接收 AFM 空间特征** | V15 通过 cross-attention 让坐标头直接看到 c_patches 空间特征，这是本项目最有价值的 AFM 特化设计 |
| **Video ViT 编码器** | 将 10 层 constant-height AFM 切片视为时间维度，3D tubelet patches 有效提取层间变化特征 |
| **坐标归一化 ÷ 12.0** | 将 24 Å cell 映射到 [-1, 1]，所有模型操作在此归一化空间进行 |

## 3. V16b 的活跃根因链

按因果顺序排列：

```
根因 1: DDIM 采样器公式错误（恒等式）
  → 去噪循环每步 x_{t-1} = x_t，不做真正的 reverse diffusion
  → 所有原子停留在初始噪声位置
  ↓
根因 2: n_atoms 没有约束生成，全 85 槽位无掩码采样
  → padding 原子被当作真实原子参与去噪
  → 训练时模型从不学习处理无效槽位
  ↓
根因 3: 训练/推理 mask 不一致
  → 训练只在 GT 有效原子上算 loss
  → 推理时 denoiser 面对全 1 mask（训练从未见过）
  ↓
叠加因素: physics_guidance 在错误状态上进一步恶化
  → 连通性约束拉伪原子向最近邻
  → 排斥力太弱无法推开密集点云
  ↓
结果: 可视化中所有原子坍缩成一团
       RMSD 看似还行（~0.42）是匈牙利匹配在归一化空间的假象
       Bond Validity 0.19, Coulomb Sim 0.01, Bottom Recall 0.02
```

## 4. 各阶段任务边界

### Phase 1：不重训，只做根因验证

**目标**：只修采样器 + mask + 评估入口，验证现有 checkpoint 的生成主链是否恢复。

**允许修改的文件**：
- `src/models/diffusion.py`（重写 sample，新增 helper）
- `src/train.py`（修改 generate 参数）
- `src/eval_phase1.py`（增加 CLI 参数）
- `src/visualize_val.py`（增加 CLI 参数）
- `src/visualize_5mol.py`（增加 CLI 参数）
- `tests/`（新增单元测试）
- `run_v16c_phase1.sh`（评估入口脚本）

**不允许做的事**：
- 不修改训练损失定义
- 不修改模型主架构
- 不启用任何 guidance

**验收标准**：
- 用现有 best checkpoint + GT count + no guidance + DDIM-50 重采样
- 对已知坍缩样本输出定量诊断（median pair distance, fraction < 0.05）
- 生成可视化图（GT vs Pred, GT/Pred/Top3）
- 明确回答：修采样器后是否不再大规模坍缩

### Phase 2：修训练/验证流程并重训

**目标**：把训练、验证、选模逻辑修正确，训练一个真正可信的 V16c。

**允许修改的文件**：
- `src/train.py`
- `src/models/diffusion.py`
- `src/models/prediction_heads.py`
- `config_v16c_debug.json` / `config_v16c_train.json`
- `run_v16c_train.sh`

**关键修改**：
- validate() 传入 epoch/stage
- best checkpoint 改按生成指标选（best_proxy.pt + best_gen.pt + latest.pt）
- 去掉类型头 /3.0
- physics_guidance 传入真实 node_mask
- bottom_z_loss 提前到 Stage 2
- ring_head 改 DETR no-object（类型 + ∅ 统一 softmax）
- 每 5 epoch 三组 sanity eval

**验收标准**：
- 不再发生几何坍缩
- Bond Validity 显著高于 V16b（> 0.19）
- Count Accuracy / MAE 有实质改善
- Type Match 改善
- Bottom Recall 脱离接近 0 的状态

### Phase 3：化学增强

**前提**：Phase 1 + Phase 2 通过。

**目标**：
- 增加显式 bond_head
- 修正 Procrustes 同参考系问题
- 清理死代码

**开始前需要先提交方案。**

### Phase 4：AFM 特化增强

**前提**：Phase 1 + Phase 2 + Phase 3 稳定。

**候选项**：
- 多尺度 c_patches
- XY / Z 坐标头解耦
- 底层原子 occupancy 辅助头
- EMA
- CFG

**开始前需要先提交方案。**

## 5. 每阶段验收标准汇总

| 阶段 | 核心验收条件 |
|------|-------------|
| Phase 1 | 修采样器后，用现有 checkpoint 重采样，原子不再坍缩成团；median pair distance 接近 GT 量级 |
| Phase 2 | 重训后三组 sanity eval 通过；Bond Validity > 0.30；Bottom Recall > 0.05；Composite 改善 |
| Phase 3 | Bond head 引入后 Bond Validity + Valence Validity 显著提升 |
| Phase 4 | AFM 特化增强带来可量化的指标改善 |

## 6. 暂不实施项列表

以下项目在 V16c 的当前阶段（Phase 1-2）中**明确不实施**：

| 项目 | 原因 |
|------|------|
| 显式 bond_head | Phase 3 内容 |
| Classifier-Free Guidance (CFG) | Phase 4 实验性内容 |
| XY / Z 坐标头解耦 | Phase 4 实验性内容 |
| 多尺度 c_patches | Phase 4 实验性内容 |
| 离散扩散类型（MiDi 风格） | 不适用于当前 AFM 条件重建任务 |
| 训练时 slot-wise type CE 改成 Hungarian 匹配监督 | 训练时 slot 对齐天然成立，不需要改 |
| MMFF94 主评估 / GFN2-xTB | Phase 4 评估增强，不阻塞主链修复 |
| shape_guidance | 当前默认未启用，修复后再评估是否需要 |

## 7. 参考实现

采样器修复对齐以下权威实现：
- lucidrains/denoising-diffusion-pytorch：`ddim_sample` 的 `time_pairs` 机制
- huggingface/diffusers：`DDIMScheduler.step()` 的 `prev_timestep` + `alpha_prod_t_prev`
- ermongroup/ddim：官方 DDIM 的 `at` / `at_next` 双索引

mask 机制对齐：
- ehoogeboom/e3_diffusion_for_molecules (EDM)：`node_mask` 贯穿训练和推理，`remove_mean_with_mask` 每步执行

ring_head 改进对齐：
- DETR (Carion et al. 2020)：learned queries + 匈牙利匹配 + no-object class

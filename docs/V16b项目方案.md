# V16b 项目方案

> 目标：在严格保留 V15 核心前提的基础上，筛选 V10-V16 中仍然合理的设计，形成一套面向 AFM 图像到 3D 分子坐标重建的可执行主线方案。

---

## 1. 不可动摇的核心前提

V16b 的根基继承自 V15，并视为不可回退：

1. 不再假设 SE(3) 等变性是必要条件。
2. 不再使用旋转增强作为主训练策略。
3. AFM 图像像素坐标与分子坐标系直接对应。
4. `coord_head` 必须直接接收 AFM 空间特征，以支持精确原子落位。

这四点是 V14 及更早版本与 V15/V16 产生性能分水岭的根因，也是 V16b 唯一必须强制继承的设计前提。

---

## 2. 历史设计筛选

### 2.1 保留到 V16b 主线的设计

| 设计 | 来源版本 | V16b 决策 | 理由 |
|---|---:|---|---|
| 固定 AFM 坐标系，不用 SE(3) 等变性 | V15 | 保留 | 与 AFM 成像物理一致，且是 V15 跃升的核心原因 |
| 关闭旋转增强 | V15 | 保留 | 旋转会破坏 AFM 像素到分子坐标的一一对应 |
| Video ViT/3D patch 编码 10 张 AFM 深度切片 | V10-V16 | 保留 | 10 层 constant-height 图像堆栈确实携带随 tip-sample distance 变化的局部对比度信息 |
| `coord_head` 对 `c_patches` 的 cross-attention | V15 | 保留 | 这是直接把 AFM 空间证据送入坐标生成的关键通道 |
| type head 基于重建干净坐标的 EDM*+γ 思路 | V14 | 保留 | 能减弱噪声坐标对类型预测的污染 |
| `type_adapter` | V14 | 保留 | 作为 type 分支的独立特征变换，比完全共享 backbone 更稳 |
| AtomCountHead | V9+ | 保留，并升级为主指标头 | 原子数是坐标解码的必要约束，也是最终任务核心指标 |
| shape descriptor 作为训练辅助目标 | V9/V15 | 保留，但只作弱辅助 | 可提供全局形状正则，但不应凌驾于局部 AFM 空间证据 |
| 物理约束 loss（键长/键角/平面性） | V14/V15 | 保留 | 有明确化学先验价值，直接服务几何合理性 |
| mild physics guidance | V8+ | 保留，但强度受控 | 有助于减少离散原子和明显不连通结构 |
| ring bond/count loss | V13/V15 | 保留 | 对含环有机分子的坐标稳定性和环质量有直接帮助 |
| `require_ring=true` 的当前实验轨道 | V13 | 保留为当前主实验设置 | 现有任务聚焦含 5/6 元环分子，且数据集大部分样本本就含环 |
| RingDetectionHead | V16 | 保留 | 其职责应是从 AFM 中预测环数量/中心/类型，而非依赖 GT 环信息 |
| 模板 snap 到标准环模板 | V16 | 保留，但仅作软约束 | 对规则芳香环的 XY 几何恢复有帮助，但不能过强、不能依赖 oracle |
| warm-start from V15 | V16 | 保留 | V16 新增 ring head，而坐标主干最好继承 V15 已收敛空间能力 |
| confusion matrix / corrugation 分组 / ring preservation / composite | V13-V15 | 保留 | 这些是评估口径，不属于主模型，但对判断真实收益必要 |

### 2.2 从 V16b 主线移除的设计

| 设计 | 来源版本 | V16b 决策 | 理由 |
|---|---:|---|---|
| SE(3)/E(3) 等变主干 | V1-V14 | 移除 | 与固定 AFM 坐标假设冲突，不再是主问题所需不变性 |
| 旋转增强 | V1-V14 | 移除 | 会直接破坏空间对齐关系 |
| 采样时 shape guidance 梯度引导 | V9/V10/V15 当前残留 | 移除 | 历史实验和 AFM 成像机理都提示其容易用全局形状先验牺牲局部原子落位 |
| TypePredictor 主线 | V11 | 移除 | 当前任务主瓶颈不是增加一个独立类型分支，而是把类型预测和空间证据真正对齐 |
| GNN 替换主线 type 预测 | V12 | 移除 | 在 V13/V14 文档中已被证明可能反向退化；可保留为实验支线，不进默认主流程 |
| default ensemble(GNN + denoiser) | V14 | 移除出主线 | 可作附加实验，但不应成为默认评估链的一部分 |
| recycling/MMFF94 主线 | V12/V13 | 移除 | 历史文档已指出在类型不稳时会恶化坐标和键距 |
| 非法 3/4 元环拆解 | V13 | 移除 | 文档明确记录会误伤正确环结构 |
| retrieval loss / retrieval head 作为主训练目标 | V9+ | 移除出主线 | 与五个核心指标关联弱，且可能分散训练容量 |
| GT ring info 参与默认推理/评估 | V8-V16 部分脚本残留 | 移除 | 会污染真实端到端结论 |
| 全局 Z 轴重权 coord loss | V13 | 移除 | 历史结果未证明全局 Z 重权有效；更合理的是保留弱的定向 Z 辅助项 |

### 2.3 作为可选辅助项保留，但不进入默认主线

| 设计 | 来源版本 | V16b 状态 | 说明 |
|---|---:|---|---|
| GNN type classifier | V12 | 可选支线 | 只用于额外消融，不作为默认推理链路 |
| denoiser+GNN ensemble | V14 | 可选支线 | 仅在单独实验中启用 |
| retrieval/CID 分析 | V9+ | 可选分析工具 | 用于检索和可视化，不作为核心损失 |
| oracle ring eval | V16 | 诊断模式 | 只能作为上界测试，不能作为默认指标 |

## 3. 结合论文与领域研究的综合判断

### 3.1 来自 AFM 研究的直接启示

1. HR-AFM 的强项是高分辨率局部结构成像，特别是原子位置、键、环等局部几何。2015 年 Nature Communications 的工作已经说明，分子内部对比主要来自 repulsive regime 下的 vdW 与 Pauli repulsion 差异，而这类信息本质上是局部、空间相关的，不是旋转不变的抽象全局特征。
2. 2024 年基于 QUAM-AFM 的 npj Computational Materials 工作证明：10 张 constant-height AFM 图像堆栈足以让模型从图像预测出 ball-and-stick 结构与化学组成，并且论文特别强调局部 patch 分析对 AFM 图像尤其有效，因为原子诱发的图像特征更依赖局部化学环境，而非远处原子。
3. 但 AFM 到完整 3D 结构的逆问题仍然困难。2021 年 PLoS Computational Biology 的 rigid-body fitting 工作明确指出，AFM 观测到的数据受限于表面高度，而且高度依赖 probe tip 形状；2024 年 Nature Reviews Materials 的综述也直接指出，传统 AFM 在复杂 3D 结构重建上会受到 incomplete surface capture 和 artifacts 的限制。
4. 分子几何学习和扩散生成研究普遍表明，分子结构质量不能只看整体坐标误差，还必须同时看局部距离、键角、类型一致性和拓扑合理性；因此联合优化坐标、类型、几何和局部结构是更稳妥的路径。

### 3.2 对 V16b 可达性的判断

综合本项目文档和上述研究，V16b 可以合理追求的目标是：

1. 对于以芳香环、准平面或中等起伏有机分子为主的 QUAM-AFM 分布，达到高质量的 XY 原子落位、正确的环拓扑和化学上合理的 3D 骨架。
2. 对顶部或可直接被 AFM 深度切片显著编码的原子，实现较低 RMSD。
3. 同时将类型预测、原子数预测和环预测作为并列目标，而不是后验修补项。
4. 对底层被遮挡原子、强非平面多环、明显悬空或高度重叠结构，仍然保持保守预期，不夸大完整精确 Z 坐标能力。

因此，V16b 的结论不应写成“AFM 图像足以稳定恢复任意分子的精确全 3D 原子坐标”，而应写成：

> 在保留 V15 固定坐标系假设并引入适量环与化学先验后，模型有望从 AFM 图像中恢复对有机小分子足够准确的 3D 结构骨架，并同时达到较高的原子类型准确率、原子数准确率和环预测质量；但对完整精确 Z 坐标，特别是底层原子和高度非平面区域，仍然需要保守表述。

### 3.3 面向五个主指标的研究归纳

V16b 的默认主线围绕以下五个并列目标设计：

1. `3D 坐标准确`
2. `原子类型准确`
3. `原子间间距/键角/局部几何合理`
4. `原子数预测准确率高、MAE 低`
5. `环数量、环中心、环类型、环保持质量高`

结合 AFM 与分子几何学习相关研究，这五个目标的合理建模方式如下：

| 目标 | 文献启示 | V16b 设计结论 |
|---|---|---|
| 3D 坐标准确 | AFM 的强信号集中在局部空间对比和表面高度；固定坐标系和局部 patch 直接建模更符合成像物理 | 必须保留 `coord_head <- c_patches` 的空间直连，不回退到旋转不变主干 |
| 原子类型准确 | QUAM-AFM 结果说明 10 张 constant-height 堆栈足以支持结构与组成联合识别，但类型信息依赖局部化学环境 | 类型分支必须同时看 `clean coords` 和局部 AFM 空间特征，不能只靠全局向量 |
| 几何合理 | 几何学习研究普遍表明 bond length、bond angle、distance matrix 是最关键的局部结构约束 | 约束不应只做推理后修补，而要进入训练主损失，且必须作用在预测结构上 |
| 原子数准确 | AFM 结构重建首先受“有多少个可解释峰/结构单元”制约；原子数错误会直接污染后续坐标和类型 | `AtomCountHead` 必须升级为主目标头，不再只是辅助头 |
| 环预测质量 | HR-AFM 对芳香环、局部闭环和键连接模式很敏感；规则环的模板先验有效，但只能作为软约束 | `RingDetectionHead + ring_loss + soft template snap` 应保留，但不得使用 GT ring info |

### 3.4 对各类设计的论文一致性判断

| 设计 | 与文献是否一致 | 结论 |
|---|---|---|
| 固定坐标 + 空间特征直达 coord_head | 高一致 | 强保留 |
| 局部 patch / 3D stack 编码 | 高一致 | 强保留 |
| 类型分支叠加局部 AFM 特征 | 高一致 | 强保留 |
| 环模板软约束 | 中高一致 | 保留，尤其适用于芳香环丰富数据 |
| 轻量化化学/物理约束 | 中高一致 | 保留 |
| 采样期强 shape guidance | 低一致 | 移除 |
| 过度依赖后置 GNN 修正主结构 | 低一致 | 不进主线 |
| 以 retrieval 作为主正则 | 证据不足 | 不进主线 |

## 4. V16b 主线方案

### 4.1 主线目标

V16b 主线优化五个并列核心指标：

> 从 10 张 AFM constant-height 图像堆栈中，同时预测足够准确的 3D 分子坐标、较高准确率的原子类型、合理的局部几何关系、准确的原子数，以及高质量的环结构。

其中：

1. `坐标准确` 是基础目标；
2. `类型准确`、`原子数准确`、`环质量高` 是并列主指标，不是附属指标；
3. `几何合理` 通过键长、键角、平面性和局部距离统计共同约束。

CID 检索、GNN 后修正、recycling、MMFF94 都降级为辅助或对照实验，不进入默认主线。

### 4.2 主线架构

```text
AFM image stack (10 x 128 x 128)
  -> Video ViT encoder
  -> c_global + c_patches
  -> Shared spatial denoiser trunk
     - coord branch receives c_patches via cross-attention
     - type branch uses EDM*+γ clean-coordinate path + type_adapter
     - local AFM patch pooling at predicted atom XY positions
  -> AtomCountHead predicts count distribution + expected count
  -> RingDetectionHead predicts ring count / centers / types
  -> Geometry auxiliary branch predicts local bond-distance / angle consistency
  -> DDIM sampling with mild physics guidance
  -> predicted RingDetectionHead outputs
  -> soft template snap for detected 5/6-membered rings
  -> final 3D coordinates
  -> final atom types
```

V16b 的关键新增要求：

1. `type branch` 不只依赖 denoiser hidden states，还应结合按预测原子 XY 位置从 `c_patches` 采样到的局部 AFM 特征。
2. `AtomCountHead` 需要同时优化分类精度和期望值误差，以同时提升 exact accuracy 和 MAE。
3. 增加轻量 `geometry auxiliary branch` 作为训练辅助头，用于预测局部 bond-distance / angle consistency，不参与默认最终推理输出，但直接约束主干学习几何合理性。
4. `RingDetectionHead` 的职责是预测 `n_rings + centers + ring_types`，再驱动 soft snap；不能把 ring template 当 oracle 输入。

### 4.3 训练损失

V16b 保留的默认损失：

1. `coord_loss`: 标准噪声预测 MSE，不做全局 Z 轴重权。
2. `type_loss`: EDM*+γ 路径上的 type CE + 类别平衡 + SNR 软加权。
3. `shape_loss`: 形状描述符辅助约束，但权重必须低于坐标、类型、原子数、环。
4. `count_ce_loss`: 原子数分类损失。
5. `count_mae_loss`: 原子数期望值校准损失。
6. `ring_loss`: 环内邻接键距与环数量相关损失。
7. `ring_det_loss`: RingDetectionHead 的环数量/中心/类型监督。
8. `constraint_loss`: Stage 2+ 启用的键长/键角/平面性弱约束，必须基于预测结构计算。
9. `geom_aux_loss`: 局部 bond-distance / angle consistency 辅助损失。
10. `bottom_z_loss`: 只作为弱辅助项保留，用于低噪声阶段拉住底部原子 Z，不替代主坐标损失。

V16b 不保留的默认损失：

1. `retrieval_loss`
2. 任何 GNN 相关主损失
3. 采样期 shape guidance 对应的额外目标

建议的默认权重：

| 损失项 | Stage 1 | Stage 2 | Stage 3 | 说明 |
|---|---:|---:|---:|---|
| `coord_loss` | 1.0 | 1.0 | 1.0 | 主损失基准 |
| `type_loss` | 0.8 | 1.0 | 1.0 | 类型为并列主目标，后期不得过低 |
| `count_ce_loss` | 0.6 | 0.7 | 0.8 | 提升 exact accuracy |
| `count_mae_loss` | 0.2 | 0.3 | 0.4 | 压低 MAE |
| `shape_loss` | 0.10 | 0.08 | 0.05 | 只做弱辅助 |
| `ring_loss` | 0.20 | 0.30 | 0.35 | 逐步增强 |
| `ring_det_loss` | 0.30 | 0.40 | 0.50 | 环为主指标之一 |
| `constraint_loss` | 0.00 | 0.08 | 0.12 | 必须基于预测结构 |
| `geom_aux_loss` | 0.00 | 0.05 | 0.08 | 学局部几何统计 |
| `bottom_z_loss` | 0.00 | 0.00 | 0.05 | 仅后期启用 |

五个主指标与损失的对应关系：

| 主指标 | 主要驱动损失 |
|---|---|
| 3D 坐标准确 | `coord_loss`, `bottom_z_loss` |
| 原子类型准确 | `type_loss` |
| 原子间距合理 | `constraint_loss`, `geom_aux_loss`, `ring_loss` |
| 原子数准确率 / MAE | `count_ce_loss`, `count_mae_loss` |
| 环预测质量 | `ring_det_loss`, `ring_loss` |

### 4.4 推理约束

默认推理只保留三类约束：

1. 轻量化 physics guidance
2. `RingDetectionHead.predict()` 产生的 predicted rings
3. predicted ring 的 soft template snap

默认推理参数建议：

1. `DDIM steps = 50` 作为默认验证配置；最终报告可额外给出 `100` 步结果。
2. `physics guidance` 只在中后期采样启用，强度保持弱约束，不允许主导坐标。
3. `ring snap` 只在低噪声阶段启用，且 `blend <= 0.25`，避免模板硬覆盖 AFM 证据。
4. 仅对高置信度 predicted rings 应用 snap，建议加入 `ring_confidence_threshold = 0.6`。
5. 默认不启用 shape guidance。

默认推理不得使用：

1. GT ring templates
2. MMFF94/recycling
3. GNN 替换默认 type
4. shape guidance 梯度引导

### 4.5 训练策略

1. 以 `V15 best checkpoint` 做 warm-start。
2. 保持 `augment_rotation=false`。
3. 默认 30 epoch，stage 边界沿用 V16：1-12 / 13-22 / 23-30。
4. Stage 1 重点学 `count + coarse coords + base type`。
5. Stage 2 重点学 `ring detection + ring geometry + bond/angle/planarity`。
6. Stage 3 重点学 `bottom Z correction + count calibration + hard-case type refinement`。
7. `warm_start` 必须使用 `strict=False`，并记录 `missing_keys / unexpected_keys`。
8. 若显存允许，优先保持 batch size 稳定，不用额外引入大幅度 curriculum 图像增强。

推荐的参数与策略调整：

| 项目 | 建议值 | 理由 |
|---|---:|---|
| `epochs` | 30 | 与现有 V16 规模兼容 |
| `ddim_steps(eval)` | 50 默认，100 复核 | 在成本和稳定性之间折中 |
| `augment_rotation` | `false` | 保持像素-坐标对齐 |
| `require_ring` | `true` 当前轨道保留 | 与当前数据分布一致 |
| `shape_loss_weight` | `<= 0.10` | 避免全局形状压制局部落位 |
| `ring_snap_blend_max` | `0.25` | 只能软修正 |
| `ring_conf_threshold` | `0.6` | 降低误 snap 风险 |
| `constraint_start_epoch` | 13 | 先学会放点，再学几何 |
| `bottom_z_start_epoch` | 23 | 后期细修 Z |

### 4.6 评估协议

默认评估必须满足：

1. 端到端调用真实 `model.generate()`。
2. 默认不允许 GT ring info。
3. `oracle_ring_info` 只能作为诊断上界单独报告。
4. 必须报告：`RMSD`、`Bottom Recall`、`Type Accuracy / Conditional Type Accuracy`、`Bond Validity`、`Count Exact Accuracy`、`Count MAE`、`Ring Preservation`、`Ring Count/Type/Center` 指标、`Composite`、`corrugation` 分组。
5. 主结果表必须先展示五个核心指标，再展示辅助指标。

## 5. 与 V16b 不一致时，代码应如何判定

若某段代码满足以下任一条件，应视为偏离 V16b：

1. 引入了旋转增强或重新回到 SE(3)/E(3) 等变主干。
2. 默认主流程使用 GT ring 信息。
3. 采样默认启用 shape guidance。
4. 默认主训练仍包含 retrieval loss。
5. GNN / recycling / MMFF94 进入默认主流程。
6. 约束损失不是作用在模型预测结构上，而是作用在 GT 上。
7. 评估脚本没有走真实端到端采样链路。
8. `AtomCountHead` 只优化分类而不校准 MAE，或评估时不报告 exact accuracy/MAE。
9. 类型分支不利用局部 AFM 空间证据，只依赖全局或纯坐标特征。
10. 环分支没有单独监督环数量/中心/类型，或默认使用 oracle ring 信息。
11. 为了提高某一项表面分数而明显牺牲其他四个核心指标。

- `src/data/dataset.py`: `require_ring`、关闭旋转增强的能力、ring 数据管线
- `src/models/video_vit.py`: 10 层图像堆栈编码
- `src/models/diffusion.py`: `coord_head` 接收 patch cross-attention；EDM*+γ clean-coordinate type path；type_adapter；ring snap 框架
- `src/models/prediction_heads.py`: AtomCountHead 与 RingDetectionHead
- `config.json`: `augment_rotation=false`、`require_ring=true`、`warm_start` 已补回

### 6.2 偏离 V16b 的关键点

1. `src/train.py`
   - 仍实例化 `TypePredictor`，但主训练和主推理并未真正使用，属于 V11 残留分支。
   - 仍把 `retrieval_loss` 纳入主训练总损失，不符合 V16b “坐标主线不带 retrieval 正则”的决策。
   - `constraint_loss` 目前是对 batch 的 GT `coords/types` 计算，而不是对预测结构计算，等价于常数项，不符合“物理约束用于约束生成结果”的设计。
   - `AtomCountHead` 还未升级为 `count classification + expected-count calibration` 双目标。
   - 类型分支还没有显式接入“按预测原子 XY 位置采样的局部 AFM 特征”这一增强路径。

2. `src/models/diffusion.py`
   - 采样时仍保留 `_apply_shape_guidance()` 调用，不符合 V16b 对 shape guidance 的移除决策。
   - `bottom_z_loss` 并未在主扩散损失中真实实现，而是文档保留、代码缺失。
   - 还没有 `geom_aux_loss` 这一面向 bond-distance / angle consistency 的辅助约束头。
   - ring snap 的启用时机和 blend 强度仍需按 V16b 收紧到“低噪声阶段 + 高置信度 predicted rings + 软约束”。

3. `src/eval_phase1.py`
   - 仍使用 `generate_v10_style()` 覆盖真实 `generate()`，默认评估不是严格的 V16/V16b 端到端路径。
   - 虽已加入 `--oracle_ring_info` 开关，但当前脚本仍绕开真实 `ring_head -> sample(predicted_rings)` 默认链路。
   - 评估输出尚未把 `Type Accuracy / Count Exact Accuracy / Count MAE / Ring Count-Type-Center` 作为主结果第一层指标。

4. `src/eval_recycling.py`
   - 仍把 GNN、ensemble、GT ring info 作为主流程的一部分，属于 V16b 明确降级为“可选支线”的内容。

5. `src/train_gnn.py`、`src/models/gnn_type_classifier.py`
   - 作为 V12 支线实验代码保留可以接受，但不属于 V16b 默认主线。

6. `src/visualize_val.py`
   - 仍保留 GNN ensemble 可视化逻辑，属于实验支线而非主线。

7. `run_v16.sh`
   - 文案仍写 50 epochs，与当前配置和 V16/V16b 的 30 epoch 设定不一致。

## 7. V16b 下一步修改优先级

### P0：必须修改

1. 让 `eval_phase1.py` 默认走真实 `model.generate()`。
2. 从主训练总损失中移除 `retrieval_loss`。
3. 修正 `constraint_loss`，使其约束对象变成预测结构而不是 GT。
4. 关闭默认采样中的 shape guidance。
5. 将 `AtomCountHead` 升级为 `count CE + expected-count MAE` 双目标。
6. 给类型分支补上基于预测原子 XY 的局部 AFM 特征采样。
7. 统一 `run_v16.sh` 与 `config.json` 的 epoch 说明。

### P1：建议修改

1. 将 `TypePredictor` 从主模型中降级为实验分支，避免误导。
2. 明确把 GNN / recycling / oracle ring eval 标注为“非主线”。
3. 重新实现或补回弱版 `bottom_z_loss`。
4. 增加轻量 `geom_aux_loss`，直接学习局部 bond-distance / angle consistency。
5. 让环模板 snap 只对高置信度 predicted rings 且只在低噪声阶段生效。

### P2：可后续处理

1. 将 retrieval/CID 相关代码迁到单独分析模块。
2. 为 V16b 单独建立 `run_v16b.sh`、`config_v16b.json`、`experiments/v16b/`。

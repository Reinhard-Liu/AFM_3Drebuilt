## 五、可视化核心观察

基于 15 张 GT vs Pred 和 15 张 GT/Pred/Top-3 对比图：

### 改善点
1. **结构不再坍缩**：V16b 的"所有原子挤成一团"问题已完全消除
2. **原子数预测接近 GT**：多数样本 pred/GT 原子数差距在 1-3 个以内
3. **XY 平面投影基本合理**：预测结构的水平展开范围与 GT 一致
4. **Type Match 达到 40-70%**：部分样本（如 #499, Type=0.71）已相当准确

### 持续问题
1. **Z 轴展开不足**：预测结构在 Z 方向明显压缩，GT 有 Z 范围 ±0.5 的结构，预测往往被压到 ±0.2 以内
2. **原子间距偏小**：预测的原子间距约为理想键长的 45-70%，导致结构整体偏紧凑
3. **颜色（类型）仍有明显错误**：尤其是 H 原子（白色）和 C 原子（黑色）经常混淆
4. **Bottom 原子几乎不可见**：Bottom Recall 仅 6%，底层原子重建极弱

## 六、V16c 最终判断

### 已解决的关键问题
1. **采样器公式错误** → 已修复（DDIM/DDPM 标准公式 + node mask + CoM removal）
2. **Bond enforcement 死逻辑** → 已修复（使用 IDEAL_BOND_LENGTHS 作为目标，梯度双向生效）
3. **Bond 标准不一致** → 已统一（constraints.py 作为 single source of truth）
4. **Bond Validity = 0** → **已突破**（从恒定 0 提升到 0.27-0.40）

### 实质提升的指标
| 指标 | V16b (debug2 E6) | V16c (best E12) | 变化 |
|------|-------------------|-----------------|------|
| Bond(gt_mask) | 0.000 | **0.395** | 从 0 突破 |
| Bond(pred_mask) | 0.000 | **0.398** | 从 0 突破 |
| Type Match | 0.095 | **0.400** | +320% |
| Count Accuracy | 0.316 | **0.309** | 持平 |
| Composite | 0.465 | **0.513** | +10% |

> 注：V16b 使用旧 Bond 口径（tolerance=30%），V16c 使用新口径（tolerance=25%），
> 因此 V16c 的 Bond=0.40 在更严格标准下取得，实际改善幅度更大。

### 仍然存在的问题
1. **Count Accuracy 停滞**（~0.31）：没有随训练显著提升
2. **Bottom Recall 极低**（0.06）：底层原子基本无法重建
3. **C_pred_guided RMSD 劣化**：guidance 把 RMSD 从 0.19（A/B）推高到 0.35
4. **Epoch 12 是峰值，后续回落**：E13-E15 的 Bond 和 Type 均下降
5. **predicted_rings 仍未接通**：C 配置不是完整 mainline

## 七、为什么 best_gen.pt（E12）优于 E15

| 对比 | E12 | E15 | 分析 |
|------|-----|-----|------|
| Bond(gt) | **0.395** | 0.282 | E12 峰值后回落 |
| Type | **0.400** | 0.347 | 同步回落 |
| Composite | **0.513** | 0.497 | 差 3% |
| RMSD | 0.360 | 0.354 | 几乎持平 |

**原因分析**：
1. **S3 constraint_weight=0.5 可能过强**：E12 达到 bond/type 的最佳平衡点后，继续施加 0.5 权重的约束反而干扰了坐标学习
2. **bond_angle_loss 主导 constraint**：bl=0.001 vs ba=0.624，bond_angle 占了约束总量的 ~83%，bond_length 几乎没有学习信号
3. **训练总 epoch 不够**：15 epoch 中只有 E10-E15（6 个 epoch）启用了约束，模型还在剧烈调整期

## 八、V16d 实验建议

### 继续保留的 V16c 正确设计
- 修正后的 DDIM/DDPM 采样器
- Node mask + CoM removal
- 统一的 IDEAL_BOND_LENGTHS / MAX_BOND_DIST
- 修正后的 `_apply_physics_guidance` bond gradient
- 双 bond metrics（gt_mask / pred_mask）
- 三组 sanity eval (A/B/C)
- best_gen.pt 按 composite score 选择

### 下一轮最优先要改的 5 个点

**1. 降低 S3 constraint_weight，延长训练周期**
- S2: 0.2, S3: 0.3（当前 0.3/0.5 过强）
- Epochs: 至少 30，最好 45
- 理由：E12 峰值后回落说明约束过早过强地压制了坐标学习

**2. bond_length_loss 需要独立加权提升**
- 当前 bond_length 在 constraint 内部权重 1.0，但实际值只有 0.001，远低于 bond_angle（0.624）
- 建议：在总 loss 中额外单独加一项 `bond_length_weight * bond_length_loss`（建议 1.0-2.0）
- 或者调整 `compute_all_constraints` 内部权重为 `5.0*bond + 0.5*angle + 0.3*planarity`
- 理由：bond_length 是 Bond Validity 的直接驱动力，当前几乎被 angle 淹没

**3. Bottom atom 加权需要更早引入**
- 当前 bottom_z_loss 只在 S2/S3 激活，且权重较弱
- 建议：从 S1 后半段（50%）就开始引入 bottom_z_loss
- 理由：Bottom Recall=6% 是当前最弱的指标

**4. Guidance 需要温和化**
- C_pred_guided 的 RMSD（0.35）显著劣于 A/B（0.19），说明 guidance 有副作用
- 建议：将 guidance step size 从 0.005 降到 0.002
- 同时在 guidance 中加入"只在后半段步骤启用"的限制（如 t < 500 才启用 bond gradient）
- 理由：在高噪声步骤施加 bond gradient 会干扰去噪

**5. 从 V16c best_gen.pt warm-start**
- 不从头训练：V16c 的权重已经学到了合理的坐标尺度和类型分布
- 从 E12 checkpoint warm-start，用更温和的约束继续训练
- 理由：节省前 9 epoch 的无约束预热期

### 先不要做的事
- 不做 bond_head（需要先稳定当前架构）
- 不做 CFG / classifier-free guidance
- 不做 XY/Z 解耦
- 不做离散扩散类型
- 不修 predicted_rings 的完整 ring snap 系统
- 不做 MMFF94/GFN2-xTB 主评估

### 关键决策建议
1. **S3 强度**：建议降低到 0.3（当前 0.5 导致 E12 后回落）
2. **C_pred_guided 是否继续作为默认推理主线**：是，但需要降低 guidance step size
3. **是否从头训练**：否，建议从 V16c best_gen.pt warm-start
4. **是否把 epochs 改回更长**：是，建议 30-45 epoch

# 环检测问题解决方案

## 问题总结

### 核心问题
验证样本（如 `val_sample_00000.png`）中，Ground Truth 有2个5元环，但预测结构中环结构不明显。

### 根本原因
1. **原子对应关系错乱**
   - RMSD=0.307 Å 使用匈牙利算法最优匹配计算
   - 该算法重新排列原子对应关系来最小化总距离
   - 例如：`Pred#0 → GT#7`（预测的第0个原子匹配到真实的第7个原子）
   - 环检测算法按索引直接对应，认为 `pred_atom_0 = gt_atom_0`，导致环内所有键断裂

2. **原子数量预测错误**
   - Ground Truth: 21个原子
   - Predicted: 36个原子（误差 +71%）
   - 引入15个虚假原子

3. **原子类型预测准确率低**
   - 仅 52.4% 准确率
   - O/N 经常被误判为 H/C

### 实验验证结果
即使假设原子类型100%正确，由于原子对应关系错乱：
- **环0**的5个键全部断裂（距离误差：1.8-7.2 Å）
- **环1**的5个键全部断裂（距离误差：4.4-11.5 Å）
- 按原始索引的 RMSD = 6.035 Å（vs 匈牙利匹配后 0.307 Å）

## 解决方案

### 方案1：基于匹配的后处理环约束（推荐：最快见效）

**核心思路**：生成后先建立原子对应关系，再在匹配空间中检测和约束环

#### 算法流程

```python
def generate(self, batch, use_gt_count=False):
    afm = batch["afm_stack"]
    c = self.encoder(afm)

    # 1. 预测原子数
    n_atoms = self.count_head.predict(c)

    # 2. 第一次生成（无环约束）
    coords, type_logits = self.ddpm.sample(c, n_atoms, max_atoms=MAX_ATOMS)
    pred_types = type_logits.argmax(dim=-1)

    # 3. 【新增】建立原子对应关系（匈牙利算法）
    if "coords" in batch:  # 验证/测试阶段
        gt_coords = batch["coords"]
        gt_mask = batch["atom_mask"]

        # 使用匈牙利算法匹配
        matched_indices = hungarian_match(coords, gt_coords, gt_mask, n_atoms)

        # 重新排列预测结果
        coords = reorder_atoms(coords, matched_indices)
        pred_types = reorder_atoms(pred_types, matched_indices)

    # 4. 【新增】基于匹配后的坐标和类型检测环
    pred_elements = [ATOM_TYPES[t] for t in pred_types[:n_atoms]]
    ring_info = detect_rings(coords[:n_atoms], pred_elements, normalized=True)

    # 5. 【新增】如果检测到环，用环约束优化坐标
    if ring_info['n_rings'] > 0:
        coords = apply_ring_constraints(
            coords, pred_types, ring_info,
            num_iterations=10, lr=0.01
        )

    return {"coords": coords, "type_logits": type_logits, ...}
```

#### 辅助函数

```python
def hungarian_match(pred_coords, gt_coords, gt_mask, n_atoms):
    """返回重新排列的索引映射"""
    from scipy.optimize import linear_sum_assignment

    n_gt = int(gt_mask.sum())
    cost_matrix = torch.cdist(pred_coords[:n_atoms], gt_coords[:n_gt])
    row_ind, col_ind = linear_sum_assignment(cost_matrix.cpu().numpy())

    return torch.from_numpy(col_ind)

def apply_ring_constraints(coords, types, ring_info, num_iterations=10, lr=0.01):
    """用梯度下降优化坐标，满足环的刚体约束"""
    coords = coords.clone().requires_grad_(True)
    optimizer = torch.optim.Adam([coords], lr=lr)

    for _ in range(num_iterations):
        loss = compute_ring_constraint_loss(coords, types, ring_info)
        optimizer.zero_grad()
        loss.backward()
        optimizer.step()

    return coords.detach()
```

#### 优缺点

**优点**：
- ✅ 实现简单，只需修改 `generate()` 方法
- ✅ 不需要重新训练模型
- ✅ 立即可以改善验证集的环检测
- ✅ 保留了现有的匈牙利匹配RMSD计算

**缺点**：
- ❌ 仅在验证/测试时有效（需要GT坐标来匹配）
- ❌ 真实推理时无法使用（没有GT参考）
- ❌ 不解决原子数量和类型预测错误的问题

#### 实现文件
- 修改：`src/train.py` - `AFM3DReconModel.generate()`

### 方案2：添加图神经网络（GNN）预测连接关系

**核心思路**：让模型直接预测原子之间的成键关系，而不依赖几何距离阈值

#### 架构改进

```python
class AFM3DReconModel(nn.Module):
    def __init__(self, config):
        super().__init__()
        # ... 现有组件 ...

        # 【新增】图预测头
        self.graph_head = GraphPredictionHead(
            node_dim=128,  # 每个原子的特征维度
            edge_dim=64,   # 边特征维度
            num_layers=3,  # GNN层数
        )

    def forward(self, batch, ...):
        # ... 现有的坐标、类型预测 ...

        # 【新增】预测邻接矩阵
        # 构建原子特征：坐标 + 类型嵌入
        atom_features = torch.cat([
            coords,  # (B, N, 3)
            F.one_hot(types, num_classes=NUM_ATOM_TYPES),  # (B, N, 10)
        ], dim=-1)

        # GNN预测成键概率
        adjacency_logits = self.graph_head(atom_features, mask)  # (B, N, N)
        adjacency_probs = torch.sigmoid(adjacency_logits)

        # 损失：与GT邻接矩阵对比
        if "gt_adjacency" in batch:
            graph_loss = F.binary_cross_entropy(
                adjacency_probs * mask.unsqueeze(-1),
                batch["gt_adjacency"]
            )
            losses["graph_loss"] = graph_loss

        # 从邻接矩阵检测环
        rings = detect_rings_from_adjacency(adjacency_probs > 0.5)

        return losses
```

### 方案3：改进原子数量预测（治标先行）

**核心问题**：当前预测36个原子，真实只有21个，误差+71%

#### 3.1 增强 AtomCountHead

```python
class ImprovedAtomCountHead(nn.Module):
    def __init__(self, embed_dim=512, max_count=85):
        super().__init__()
        self.max_count = max_count

        # 【改进1】更深的网络
        self.encoder = nn.Sequential(
            nn.Linear(embed_dim, 512),
            nn.LayerNorm(512),
            nn.GELU(),
            nn.Dropout(0.1),
            nn.Linear(512, 256),
            nn.LayerNorm(256),
            nn.GELU(),
            nn.Dropout(0.1),
        )

        # 【改进2】多头预测
        self.cls_head = nn.Linear(256, max_count)
        self.reg_head = nn.Linear(256, 1)
        self.uncertainty_head = nn.Linear(256, 1)  # 预测不确定性

    def compute_loss(self, c, n_atoms):
        features = self.encoder(c)

        # 【改进3】Focal Loss 处理类别不平衡
        cls_logits = self.cls_head(features)
        cls_target = (n_atoms - 1).long().clamp(0, self.max_count - 1)

        # Focal Loss
        ce_loss = F.cross_entropy(cls_logits, cls_target, reduction='none')
        pt = torch.exp(-ce_loss)
        focal_loss = ((1 - pt) ** 2) * ce_loss  # gamma=2.0
        cls_loss = focal_loss.mean()

        # 【改进4】Smooth L1 回归损失
        reg_value = self.reg_head(features).squeeze(-1)
        reg_loss = F.smooth_l1_loss(reg_value, n_atoms.float())

        # 【改进5】不确定性加权
        log_uncertainty = self.uncertainty_head(features).squeeze(-1)
        precision = torch.exp(-log_uncertainty)
        count_loss = precision * (cls_loss + 0.5 * reg_loss) + log_uncertainty

        return {
            "count_loss": count_loss.mean(),
            "cls_loss": cls_loss,
            "reg_loss": reg_loss,
        }
```

### 方案4：基于置换等变性的扩散模型（根本解决）

**核心思路**：让扩散模型输出**规范化顺序**的原子，而不是无序集合

#### 架构设计

```python
class CanonicalOrderedDDPM(nn.Module):
    """输出具有规范顺序的原子序列"""

    def __init__(self, ...):
        super().__init__()
        self.denoiser = SE3EquivariantDenoiser(...)

        # 【新增】排序网络
        self.sort_key_predictor = nn.Sequential(
            nn.Linear(cond_dim + 3, 128),  # 坐标 + 条件
            nn.GELU(),
            nn.Linear(128, 1),  # 输出排序键
        )

    def sample(self, c, n_atoms, ...):
        # 标准DDPM采样
        coords = torch.randn(B, n_atoms, 3, device=device)

        for t in reversed(range(self.timesteps)):
            # ... 标准去噪步骤 ...
            pass

        # 【关键】预测每个原子的排序键
        sort_keys = self.sort_key_predictor(
            torch.cat([coords, c.unsqueeze(1).expand(-1, n_atoms, -1)], dim=-1)
        )  # (B, N, 1)

        # 按排序键重新排列原子
        sorted_indices = torch.argsort(sort_keys.squeeze(-1), dim=1)
        coords = torch.gather(
            coords, 1, sorted_indices.unsqueeze(-1).expand(-1, -1, 3)
        )
        type_logits = torch.gather(
            type_logits, 1, sorted_indices.unsqueeze(-1).expand(-1, -1, NUM_TYPES)
        )

        return coords, type_logits
```

#### 训练损失


### 方案5：迭代优化法（推理时环约束）

**核心思路**：在DDPM采样过程中动态检测和约束环

#### 实现

```python
def sample_with_ring_constraints(self, c, n_atoms, ...):
    B = c.shape[0]
    device = c.device

    # 初始化
    x_t = torch.randn(B, max_atoms, 3, device=device)
    mask = build_mask(n_atoms, max_atoms)

    for t_idx in reversed(range(self.timesteps)):
        # 标准DDPM步骤
        eps_pred, type_logits = self.denoiser(x_t, t, c, mask)
        x_t = ddpm_update(x_t, eps_pred, t_idx)

        # 【关键】每50步重新检测和约束环
        if t_idx % 50 == 0 and t_idx < 800:
            # 预测当前原子类型
            pred_types = type_logits.argmax(dim=-1)

            # 转换为元素列表
            pred_elements = [ATOM_TYPES[t.item()] for t in pred_types[0, :n_atoms[0]]]

            # 检测环（在当前噪声坐标上）
            ring_info = detect_rings(
                x_t[0, :n_atoms[0]].cpu().numpy(),
                pred_elements,
                normalized=True
            )

            # 应用环约束投影
            if ring_info['n_rings'] > 0:
                x_t = project_to_ring_constraints(
                    x_t, ring_info, strength=1.0 - t_idx / self.timesteps
                )

    return x_t, type_logits

def project_to_ring_constraints(coords, ring_info, strength=1.0):
    """将坐标投影到环约束流形上"""
    coords_opt = coords.clone()

    for ring_idx in range(ring_info['n_rings']):
        if not ring_info['ring_valid'][ring_idx]:
            continue

        # 获取环原子索引
        atom_indices = ring_info['ring_atom_indices'][ring_idx]
        valid_atoms = atom_indices[atom_indices >= 0]

        # Procrustes对齐到标准环模板
        ring_coords = coords_opt[0, valid_atoms]
        template = torch.from_numpy(ring_info['ring_templates'][ring_idx])

        # ... Procrustes对齐算法 ...
        aligned_coords = procrustes_align(ring_coords, template)

        # 混合原始坐标和对齐坐标
        coords_opt[0, valid_atoms] = (
            strength * aligned_coords + (1 - strength) * ring_coords
        )

    return coords_opt
```

#### 优缺点

**优点**：
- ✅ 不需要重新训练
- ✅ 真实推理时有效
- ✅ 直接改善环结构质量

**缺点**：
- ❌ 推理速度变慢（每50步检测一次环）
- ❌ 早期步骤噪声太大，环检测可能不准确
- ❌ 需要精心调整检测频率和约束强度
